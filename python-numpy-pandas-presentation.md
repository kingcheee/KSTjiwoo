## 🗓️ Day 1 — 5월 27일: 파이썬 기초

### 💡 왜 배우는가?
> "데이터 분석의 꽃은 NumPy야. 근데 그 꽃을 피우려면 파이썬이라는 흙이 단단해야 해."
> — 교수님의 큰 그림: 파이썬 기초 → NumPy → Pandas → 머신러닝

---

### 1. 형변환 (Type Casting)

**비유: 게임 캐릭터 전직** 🎮

키보드로 입력한 숫자는 컴퓨터가 **글자(문자열)**로 인식해.
수학 계산을 하려면 **숫자(정수형)**으로 "전직"시켜야 해.

```python
# ❌ 이러면 글자 붙이기만 됨
age = input("나이? ")        # "25" (글자)
print(age + 1)               # 에러!

# ✅ 숫자로 전직 후 계산
age = int(input("나이? "))   # 25 (숫자)
print(age + 1)               # 26
```

---

### 2. f-string 포매팅

**비유: 빈칸 채우기 놀이 (Mad Libs)** ✏️

문장 중간에 변수를 쏙쏙 끼워넣는 기능!

```python
name = "규연"
score = 95

# ✅ f-string: 가장 많이 쓰이는 방식!
print(f"안녕! {name}야. 점수는 {score}점이야!")
# → 안녕! 규연야. 점수는 95점이야!
```

> 🔥 **교수님 강조**: "현업이나 AI 프롬프트 작성할 때 f-string이 압도적으로 많이 써. 이것 하나만큼은 확실히 외워."

---

### 3. 조건문 (if / elif / else)

**비유: 용돈 맞춤형 중국집 자판기** 🥡

```python
money = 15000

if money >= 20000:
    print("탕수육! 🍖")
elif money >= 10000:
    print("짬뽕! 🍜")
else:
    print("단무지만... 🥒")
```

**자판기가 돈을 확인하고 메뉴를 결정하듯**, 컴퓨터가 조건을 판단해서 다른 행동을 해.

---

### 4. 반복문 (while / for)

**비유: 무한 달리기 vs 트랙 자동 머신** 🏃

```python
# while: "조건 만족할 때까지 계속 달려라!"
count = 0
while count < 3:
    print(f"{count}번째 반복!")
    count += 1

# for: "딱 100바퀴만 달려라!" ← 횟수 정할 때 더 편함
for i in range(3):
    print(f"{i}번째 반복!")
```

---

### 5. ⭐ 리스트 컴프리헨션 (List Comprehension)

**비유: 최첨단 스마트 컨베이어 벨트** 🏭

**기존 방식** (4줄):
```python
result = []
for x in range(10):
    if x % 2 == 0:
        result.append(x * 2)
```

**리스트 컴프리헨션** (1줄!):
```python
result = [x * 2 for x in range(10) if x % 2 == 0]
# → [0, 4, 8, 12, 16]
```

> 🔥🔥🔥 **교수님 최대 강조**: "오늘 수업에서 다른 건 몰라도 **이것 하나만 건지면 성공**이에요!"
> "무조건 써야 할 정도로 많이 쓰여. 여러 줄을 한 줄로 우아하게 줄여주거든."

**왜 중요?**
- 코드가 간결해지면 **유지보수 비용**이 줄어들어
- 현업에서 소프트웨어 10년 가까이 유지보수하는데, 코드 줄이기가 핵심!

## 🗓️ Day 2 — 5월 28일: 파일 입출력 & NumPy 기초

### 💡 왜 배우는가?
> 컴퓨터를 끄면 파이썬 데이터는 전부 날아가. 영구 저장이 필요하고, 외부 데이터를 불러와야 해. 그리고 수만 개 숫자를 초고속으로 계산하려면 NumPy가 필요해!

---

### 1. 파일 입출력 (open / with)

**비유: 도서관 책 대출 & 마법의 자동 반납기** 📚

```python
# ❌ 수동 반납 — 까먹으면 에러!
f = open("data.txt", "r")
content = f.read()
f.close()  # 이거 깜빡하면 메모리 누수!

# ✅ with문 — 마법의 자동 반납 독서실
with open("data.txt", "r") as f:
    content = f.read()
# 밖으로 나가면 자동으로 닫힘!
```

| 모드 | 의미 |
|------|------|
| `"r"` | 읽기 (Read) |
| `"w"` | 쓰기 (Write) — 덮어쓰기! |
| `"a"` | 추가 (Append) |

---

### 2. NumPy 배열 (ndarray)

**비유: 잡동사니 이사 박스 vs 계란판** 🥚

| 파이썬 리스트 | NumPy 배열 |
|-------------|-----------|
| 아무거나 막 담음 | 숫자만 담음 |
| 크기 제각각 | 딱 맞는 칸 |
| 계산 느림 | **초고속 계산** |

```python
import numpy as np

# 리스트: 느림
a = [1, 2, 3]
b = [4, 5, 6]
# a + b → [1, 2, 3, 4, 5, 6] (붙이기만 함!)

# NumPy 배열: 진짜 덧셈!
a = np.array([1, 2, 3])
b = np.array([4, 5, 6])
a + b  # → [5, 7, 9] (요소별 계산!)
```

**왜 빠른가?**
- 계란판처럼 데이터가 규칙적으로 정렬되어 있어서
- **한 번에(병렬로) 수만 개를 동시에 계산** 가능!
- 이게 머신러닝 수학의 기초 공사야

## 🗓️ Day 3 — 5월 29일: NumPy 마스크 & Pandas 기초

### 💡 왜 배우는가?
> 현실의 데이터는 표(엑셀) 형태야. 그 표를 다루는 도구가 Pandas!
> NumPy가 엔진이라면, Pandas가 우리가 직접 운전하는 자동차.

---

### 1. NumPy 마스크 필터링 (Boolean Indexing)

**비유: 원하는 것만 골라내는 마법의 투시 안경** 🥽

```python
import numpy as np

scores = np.array([85, 92, 78, 95, 60])

# 마스크 만들기: 90점 이상만 True
mask = scores >= 90
# → [False, True, False, True, False]

# 마스크 씌우기
scores[mask]
# → [92, 95]  ← 90점 이상만 쏙!
```

**실전 예시**: "스팸 여부가 0인 사람 = 정상 고객"만 걸러내기

---

### 2. Pandas DataFrame

**비유: 파이썬 세상으로 들어온 초강력 엑셀** 📊

```python
import pandas as pd

# CSV 파일 불러오기 (도서관 창고 → 내 책상 위)
df = pd.read_csv("netflix.csv")

# 데이터 훑어보기
df.shape    # (행 개수, 열 개수)
df.info()   # 전체 구조 + 빈칸 여부
df.head()   # 위에서 5개만 보기
```

> 교수님: "그냥 엑셀 표라고 생각하면 돼요. 어렵지 않아요!"

---

### 3. NumPy vs Pandas 역할 구분

| NumPy | Pandas |
|-------|--------|
| 수치 계산 엔진 | 표 데이터 다루기 |
| 머신러닝 바탕 | 전처리/분석/시각화 |
| 보이지 않는 곳에서 일함 | **우리가 직접 만지는 도구** |

## 🗓️ Day 4 — 6월 1일: Pandas 데이터 전처리

### 💡 왜 배우는가?
> "분석 전, 먼저 스캔하고 훑어봐!"
> 택배가 오면 상태 확인하듯, 데이터도 먼저 검사해야 해.
> 그리고 빈칸(결측치)은 반드시 처리해야 머신러닝에 넣을 수 있어!

---

### 1. 데이터 탐색 (택배 언박싱)

```python
df.head()    # 위 5개 — "어떻게 생겼나?"
df.info()    # 전체 구조 — "빈칸은 없나?"
df.shape     # (행, 열) — "총 몇 개지?"
```

---

### 2. 결측치 처리 (구멍 난 튜브 수리 vs 폐기) 🔧

```python
# 구멍 찾기
df.isna().sum()
# → 나이: 5개 빈칸
# → 점수: 2개 빈칸

# 방법 1: 대체 (fillna) — 테이프로 때우기
df["나이"].fillna(df["나이"].mean(), inplace=True)

# 방법 2: 제거 (dropna) — 튜브 버리기
df.dropna(subset=["점수"], inplace=True)
```

> ⚠️ **inplace=True**: "원본 파일을 손대겠다"는 의미!
> 원본이 영구 수정되니까 조심해서 써야 해.

---

### 3. ⭐ 대괄호 두 개 `[[]]`

**교수님 극강 강조**: "판다스에서 가장 많이 하실 실수! **무조건 기억해야 돼!**"

```python
# ❌ 한 개: Series (1차원) — 보기 불편
df["이름"]

# ⭐ 두 개: DataFrame (2차원 표) — 보기 좋음!
df[["이름", "나이", "점수"]]
```

---

### 4. 조건 필터링 & query()

**비유: 놀이공원 키 제한 통과대** 🎢

```python
# 방법 1: 마스크 방식
df[df["나이"] >= 20]

# 방법 2: query() — SQL처럼 직관적!
df.query("나이 >= 20")
df.query("nclass == 1 and 점수 > 80")
```

> 교수님: "SQL 개발자를 위해 만들어진 함수 아닐까?"

---

### 5. ⭐ value_counts()

**교수님 강조**: "정말 중요한 함수입니다!"

```python
# 각 값이 몇 개씩 있는지 세기
df["장르"].value_counts()
# → 액션: 150개
# → 호러: 80개
# → 코미디: 120개
```

**시각화의 시작이야!** 그래프 그리려면 개수부터 세야지.

## 🗓️ Day 5 — 6월 2일: 실전 프로젝트 (넷플릭스 데이터)

### 💡 왜 배우는가?
> 지금까지 배운 걸 전부 합쳐서 **실제 데이터 분석의 전체 프로세스**를 경험!
> 1단계: 데이터 훑기 → 2단계: 결측치 처리 → 3단계: 피처 엔지니어링 → 4단계: 시각화

---

### 1. 피처 엔지니어링 (Feature Engineering)

**비유: 해외 직구 스마트폰 한국 패치하기** 📱

```python
# 미국 등급 → 한국 연령대로 변환
rating_map = {
    "PG-13": "13세 이상",
    "R": "19세 이상",
    "TV-MA": "19세 이상"
}
df["age_group"] = df["rating"].map(rating_map)
```

**기존 데이터를 내 목적에 맞게 재가공**해서 새로운 정보를 만들어내는 거야.

---

### 2. ⭐ str.split & stack (고난도 전처리)

**비유: 햄버거 세트메뉴 해체해서 낱개로 줄 세우기** 🍔

```python
# 원본: 한 칸에 여러 장르가 섞여 있음
# "액션, 호러, 스릴러"

# Step 1: 쉼표로 분리 (세트메뉴 펼치기)
genres = df["listed_in"].str.split(", ", expand=True)

# Step 2: 한 줄로 쌓기 (낱개로 줄 세우기)
genres_stacked = genres.stack()

# Step 3: 개수 세기
genres_stacked.value_counts()
# → Drama: 1200개
# → Comedy: 800개
# → Action: 600개
```

> 🔥 **교수님 강조**: "처음이라면 좀 빡세요! 이런 케이스는 노트에 꼭 메모해 놓으세요!"
> 현업 데이터는 절대 예쁘게 주어지지 않아.

---

### 3. 시각화

#### Matplotlib 파이차트 — "피자 조각 나누기" 🍕

```python
import matplotlib.pyplot as plt

type_counts = df["type"].value_counts()
plt.pie(type_counts, labels=type_counts.index, autopct="%1.1f%%")
plt.title("Netflix: Movie vs TV Show")
plt.show()
```

**언제 쓸까?** 전체에서 각 부분의 **비율**을 볼 때!

#### Seaborn 수평 막대그래프 — "키 순서 줄 세우기" 📊

```python
import seaborn as sns

genre_counts = genres_stacked.value_counts().head(10)
sns.barplot(x=genre_counts.values, y=genre_counts.index,
            palette="coolwarm")
plt.title("Top 10 Netflix Genres")
plt.show()
```

**언제 쓸까?** 종류가 많을 때 **순위**를 볼 때!
Seaborn은 Matplotlib에 **인스타그램 필터**를 입힌 거라고 생각해.
