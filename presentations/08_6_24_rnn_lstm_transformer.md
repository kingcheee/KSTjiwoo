## Part 1. 시퀀스 데이터 — "순서가 왜 중요할까?" (5분)

### 🤔 왜 배우는가?

우리가 일상에서 다루는 데이터 중에 **순서가 바뀌면 의미가 완전히 달라지는 것들**이 있어. 문장, 음성, 주식 차트, 센서 데이터... 이런 것들을 AI가 이해하려면 먼저 **"순서"라는 개념**을 알아야 해.

### 핵심 개념

**시퀀스 데이터(Sequence Data)**: 시간의 흐름에 따라 순서대로 생성되는 데이터

💡 **비유**: 수능 국어 지문 읽기!
- "나는 학교에 간다" → 의미가 통해
- "간다 학교 나는" → 도대체 뭔 소리야?
- 단어의 **순서**가 바뀌면 의미가 완전히 달라져
- 마치 등장인물이 계속 바뀌는 두꺼운 러시아 소설을 읽는 것처럼, **앞에서부터 순서대로 읽어야** 전체 맥락을 이해할 수 있어

**시퀀스 데이터의 3가지 특성** ⭐
1. **순차적 생성**: 데이터가 시간 순서대로 만들어짐
2. **의존성(Dependency)**: 이전 데이터가 현재 데이터에 영향을 미침
3. **불변성**: 과거 데이터는 업데이트되지 않고 고정됨

### 실제 시퀀스 데이터 예시

| 유형 | 예시 | 왜 순서가 중요? |
|------|------|----------------|
| **텍스트** | 문장, 문서 | 앞 단어를 기억해야 다음 단어 의미 파악 |
| **음성** | 음성 파형, 오디오 | 시간 순서대로 소리를 분석해야 음성 인식 |
| **금융** | 주식 차트, 거래 기록 | 과거 패턴으로 미래 가격 예측, 이상 거래 탐지 |
| **센서** | IoT 센서, 네트워크 트래픽 | 시간별 정상 패턴 모니터링, 이상 감지 |

### ⭐ 교수님 강조 포인트
> "데이터가 순서대로 생성되고, 과거 데이터가 현재에 영향을 미치는 특성을 가진 데이터를 시퀀스 데이터라고 합니다. 이 순서를 지켜야만 의미를 파악할 수 있어요!"

## Part 2. RNN (순환 신경망) — "기억하는 신경망" (7분)

### 🤔 왜 배우는가?

일반 신경망은 입력을 독립적으로 처리해. 하지만 시퀀스 데이터는 **이전 정보를 기억해야** 의미를 파악할 수 있어. 그래서 등장한 게 RNN!

### 핵심 개념

**RNN의 작동 원리** ⭐

💡 **비유**: 친구에게 긴 이야기를 순서대로 전해주는 것!
- 첫 번째 친구가 두 번째 친구에게 이야기를 전하고
- 두 번째 친구는 들은 내용 + 새로운 내용을 세 번째 친구에게 전하고
- 이렇게 계속 전달하면서 **이전 내용을 기억**하는 구조

**수식으로 보면:**
- 현재 Hidden State: `h_t = tanh(W_x · x_t + W_h · h_{t-1})`
- 현재 입력(x_t) + 이전 Hidden State(h_{t-1}) → 현재 Hidden State(h_t)

**RNN의 핵심 특징:**
1. **순차적 처리**: 데이터를 시간 순서대로 하나씩 입력
2. **Hidden State(은닉 상태)**: 과거 정보를 압축해서 저장하는 단기 기억 공간
3. **파라미터 공유**: 모든 시간 단계에서 **동일한 가중치(W_x, W_h)**를 사용 → 가변 길이 입력 처리 가능

### RNN의 5가지 아키텍처 유형

| 유형 | 구조 | 비유 | 실제 활용 |
|------|------|------|-----------|
| **One-to-One** | 입력 1 → 출력 1 | 동전 하나 넣으면 캔 음료 하나 | 이미지 분류 |
| **One-to-Many** | 입력 1 → 출력 여러 개 | "봄"이라는 단어 하나로 소설 한 편 쓰기 | 이미지 캡셔닝, 작곡/작사 |
| **Many-to-One** | 입력 여러 개 → 출력 1 | 긴 영화 리뷰 읽고 "긍정/부정" 도장 | 감성 분석, 스팸 탐지 |
| **Many-to-Many (지연형)** | 입력 여러 개 → 출력 여러 개 | 통역사가 다 듣고 나서 번역하기 | 기계 번역, 챗봇 |
| **Many-to-Many (실시간)** | 입력마다 즉시 출력 | 영화 각 장면마다 실시간 자막 달기 | 품사 태깅, 개체명 인식 |

### RNN의 한계: 기울기 소실 문제 ⭐⭐

💡 **비유**: 아주 긴 수능 국어 지문을 혼자서 처음부터 끝까지 읽기
- 뒤로 갈수록 앞 페이지 내용을 까먹어
- 주인공 이름이 뭐였는지, 무슨 일이 있었는지 희미해져
- 문장이 길어지면 **맨 처음 단어의 정보가 사라져버림**

이게 바로 **기울기 소실(Vanishing Gradient)** 문제!
- 역전파할 때 기울기가 점점 작아져서 → 앞부분의 정보가 학습이 안 됨
- 결과: RNN은 **긴 문장의 맥락을 제대로 이해 못 함**

### ⭐ 교수님 강조 포인트
> "RNN은 데이터의 앞뒤 순서를 기반으로 문장의 '맥락'을 이해하기 위한 모델입니다. 단어가 들어올 때마다 이전 단어의 정보가 축적된 Hidden State 값을 계속 다음으로 넘겨줘요."

## Part 3. LSTM — "장기 기억을 가진 똑똑한 신경망" (7분)

### 🤔 왜 배우는가?

RNN의 기울기 소실 문제를 해결하기 위해 등장한 게 **LSTM**이야. 단기 기억뿐 아니라 **장기 기억**을 따로 관리하는 구조야!

### 핵심 개념

**Cell State (셀 상태)** ⭐⭐ — 장기 기억 보관소

💡 **비유**: 절대 안 찢어지는 비밀 노트!
- RNN은 Hidden State 하나에 모든 정보를 압축했기 때문에 금방 넘침
- LSTM은 **Cell State라는 별도의 장기 기억 공간**을 만들어서 중요한 정보를 오래 보존
- 마치 우리 뇌에도 단기 기억과 장기 기억이 따로 있듯이!

**3가지 게이트 (Gates)** — 수도꼭지(밸브) 비유 💡

교수님이 수도꼭지(밸브)에 비유하셨는데, 물을 틀고 잡그는 원리로 정보 흐름을 조절해.

**① 망각 게이트 (Forget Gate) — "스마트 지우개 밸브"**
- 역할: 기존 장기 기억 중 **더 이상 필요 없는 정보를 지움**
- 비유: 노트에 모든 걸 다 적으면 꽉 차서 엉망이 되니까, 퇴장한 조연 이름을 쓱쓱 지우는 지우개
- 시그모이드(σ) 함수로 0~1 사이 값 출력 → 0이면 "완전 잊어", 1이면 "완전 기억해"

**② 업데이트 게이트 (Update Gate) — "핵심 요약 형광펜 밸브"**
- 역할: 새로운 정보 중 **장기 기억으로 저장할 중요한 내용을 추가**
- 비유: 수많은 새 정보 중 "이건 진짜 중요하니까!" 하고 형광펜으로 칠하는 것
- 시그모이드(σ) + Tanh 함수로 새로운 후보 값을 생성

**③ 출력 게이트 (Output Gate) — "맞춤형 스피커 밸브"**
- 역할: 장기 기억 중 **지금 당장 출력할 정보를 결정**
- 비유: 비밀 노트 속 방대한 정보 중 "지금 말해야 할 것"만 꺼내서 콸콸 내보내는 수도꼭지
- 시그모이드(σ)로 출력할 비율을 결정

### LSTM 데이터 흐름 정리

```
입력(x_t) + 이전 Hidden State(h_{t-1})
    ↓
┌─────────────────────────────────┐
│  Forget Gate: 불필요한 기억 삭제  │ → Cell State 업데이트
│  Update Gate: 새로운 핵심 추가    │ → Cell State 업데이트
│  Output Gate: 출력할 것 결정     │ → Hidden State 생성
└─────────────────────────────────┘
    ↓
새로운 Cell State(c_t) + Hidden State(h_t)
```

### ⭐ 교수님 강조 포인트
> "LSTM은 기존 RNN의 Hidden State에 더해, **장기 기억(Long-Term Memory)을 전담하는 Cell State가 새롭게 존재**한다는 것을 꼭 기억하세요! 3개의 밸브(게이트)를 열고 닫으면서 필요 없는 기억은 버리고, 새로운 핵심은 추가하고, 필요한 것만 출력하는 똑똑한 조절을 합니다."

## Part 4. Transformer — "순서를 무시한 혁명" (5분)

### 🤔 왜 배우는가?

RNN/LSTM은 순서대로 하나씩 읽어야 해서 **느리고**, 병렬 처리가 불가능해. Transformer는 이 한계를 완전히 깨뜨렸어!

### 핵심 개념

**RNN vs Transformer — 병렬 처리의 차이** ⭐⭐

💡 **비유**: 두꺼운 책 읽기!
- **RNN**: 혼자서 처음부터 끝까지 순서대로 읽기 → 느리고, 앞부분 까먹음
- **Transformer**: 친구 1,000명을 불러서 **각자 한 페이지씩 동시에 읽기** → 순식간에 완료!

이게 바로 **병렬 처리(Parallel Processing)**!

**왜 이게 대단한가? — GPU 활용** ⭐
- AI 학습에 쓰이는 GPU는 동시 작업에 특화된 수만 명의 일꾼 같은 거야
- RNN은 데이터를 하나씩만 던져주니까 비싸고 좋은 GPU가 놀고 있어 → 슈퍼컴퓨터로 테트리스 하는 것과 같아!
- Transformer는 데이터를 한 번에 왕창 쏟아부어서 **GPU 능력을 100% 활용** → 연산 속도 폭발!

**Self-Attention (셀프 어텐션)** 💡

💡 **비유**: 탐정의 사건 보드판과 붉은 실
- 범죄 영화에서 탐정이 벽에 용의자 사진과 단서를 한 번에 쫙 붙여놓고
- 서로 관련이 깊은 것끼리 **붉은 실로 굵게 연결**하는 장면 있잖아?
- Self-Attention도 비슷해! 문장 안의 모든 단어를 한 번에 펼쳐놓고
- 각 단어가 다른 모든 단어와 **얼마나 관련이 있는지 점수(Attention Score)**를 계산해
- "I am a student"에서 'I'가 'student'와 얼마나 관계 깊은지 수학적으로 계산!

### ⭐ 교수님 강조 포인트 (시험 출제 예고!)
> "쉬운 내용인데 시험 문제 낼까요? **왜 LSTM 대신 Transformer를 써야 하는가?** → RNN/LSTM은 순서대로 하나씩 읽어서 **병렬 연산이 불가능**하기 때문입니다. Transformer는 모든 단어를 한 번에 입력받아 GPU의 병렬 처리 능력을 100% 활용할 수 있어요!"

## Part 5. PyTorch 실습 — "코드로 만드는 RNN" (3분)

### 🤔 왜 배우는가?

이론만 알고 코드를 못 짜면 의미가 없지! 실제로 PyTorch로 RNN 모델을 만들어보는 시간이었어.

### 핵심 코드

```python
# 1. 라이브러리 임포트
import torch
import torch.nn as nn
import torch.nn.functional as F

# 2. 하이퍼파라미터 설정
vocab_size = 1000    # 어휘 사전 크기
batch_size = 4       # 배치 크기
seq_len = 60         # 시퀀스 길이
embed_dim = 128      # 임베딩 차원
hidden_dim = 256     # 은닉 상태 차원

# 3. 임베딩 레이어 — 토큰 ID를 밀집 벡터로 변환
embedding = nn.Embedding(vocab_size, embed_dim)

# 4. 샘플 데이터 생성
sample_token_ids = torch.randint(low=0, high=vocab_size,
                                  size=(batch_size, seq_len),
                                  dtype=torch.long)
# Shape: (4, 60) → 4개 문장, 각 60개 토큰

# 5. 임베딩 적용
embedded = embedding(sample_token_ids)
# Shape: (4, 60, 128) → 각 토큰이 128차원 벡터로 변환!

# 6. RNN 모델 정의
rnn_model = nn.RNN(
    input_size=embed_dim,    # 입력 크기 = 임베딩 차원
    hidden_size=hidden_dim,  # 은닉 상태 크기
    batch_first=True         # 입력 형식: (batch, seq, feature)
)
```

### PyTorch 모델 만들기 핵심 규칙 ⭐

교수님이 신신당부하신 코딩 규칙:

**`__init__` 함수:**
- `self`를 반드시 넣어야 함
- `super()` 함수를 사용해 부모 클래스를 먼저 호출(상속)해야 함

**`forward` 함수:**
- 모델이 어떻게 흘러갈지 길을 만들어주는 함수
- 직접 `forward()`를 호출할 필요 없이, 객체에 데이터를 던져주면 **자동으로 호출**됨

### ⭐ 교수님 강조 포인트
> "PyTorch로 신경망 클래스를 만들 때 `__init__` 안에 `self` 넣고, `super()`로 부모 호출하는 것 꼭 기억하세요! 그리고 `forward` 함수는 우리가 직접 부르는 게 아니라 자동으로 호출됩니다."

## 🎯 핵심 요약 (3줄)

1. **시퀀스 데이터**는 순서가 의미를 결정하는 데이터 (텍스트, 음성, 금융, 센서)
2. **RNN → LSTM → Transformer**로 발전하면서, 기억력 문제를 해결하고 병렬 처리로 속도를 극대화
3. **Transformer**는 Self-Attention + 병렬 처리로 GPU를 100% 활용할 수 있어서 현대 NLP의 표준

## ❓ 예상 질문

1. **"오늘 배운 내용을 한 줄로 설명하면?"**
   → 순서가 있는 데이터를 처리하는 방식이 RNN의 순차적 처리에서 Transformer의 병렬 처리로 진화했다!

2. **"이 개념이 없으면 어떤 불편함이 생길까?"**
   → AI가 문장의 맥락을 이해하지 못해서 번역, 챗봇, 음성 인식 같은 게 모두 안 됨. 우리가 매일 쓰는 번역 앱, AI 비서 같은 서비스가 존재할 수 없어!

3. **"내 삶에 어떻게 적용할 수 있을까?"**
   → 주가 예측(금융 데이터), 이상 거래 탐지(센서 데이터), 텍스트 분석(감성 분석) 등 실제 데이터 분석 프로젝트에 바로 활용할 수 있어. 특히 시계열 데이터가 있는 분야라면 RNN/LSTM이 여전히 강력한 도구야!
