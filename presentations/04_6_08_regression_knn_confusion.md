## Part 1. 머신러닝 기본 패러다임 복습 (5분)

### 🤔 왜 배우는가?

머신러닝 알고리즘이 바뀌어도 **전체 흐름은 항상 같아**. 이 패러다임만 잘 잡으면 어떤 모델이든 다룰 수 있어!

### 핵심 개념

**머신러닝 모델링 7단계** ⭐ (교수님 반복 강조)
1. 라이브러리 임포트
2. 데이터 로드
3. 데이터 탐색 (EDA)
4. Train/Test 분할
5. 모델 생성
6. 훈련 (fit)
7. 예측 및 성능 평가

> 교수님: "이 흐름만 잘 기억하고 있으면 수학적인 내용을 완벽히 몰라도 기본적인 머신러닝 모델을 훈련하는 데 큰 어려움이 없어요!"

**Train/Test 분리 이유** ⭐ (교수님 퀴즈로 강조)
- 💡 **비유**: 문제집 풀기와 진짜 기말고사!
- 학원에서 푼 문제집(Train)과 기말고사(Test)가 똑같이 나오면 답을 달달 외워서 100점 맞는 것과 같아
- **새로운 데이터를 처음 보더라도 낯설어하지 않고 정확하게 예측할 수 있는 진짜 실력**을 기르기 위해 반드시 분리!

**선형회귀 y = wx + b** ⭐
- **W(기울기/가중치)**: x가 1 증가할 때 y가 얼마나 변하는지
- **b(절편/편향)**: x=0일 때 y의 시작값
- 교수님: "다음 주 딥러닝 수업에서 가장 기본이 되는 알고리즘! 그때 처음 듣는다고 하면 정말 곤란해져요!"

### 핵심 코드


## Part 2. KNN 분류 알고리즘 (7분)

### 🤔 왜 배우는가?

지금까지는 **회귀(숫자 예측)**를 배웠잖아. 이번엔 **분류(카테고리 예측)**를 배워보자!

### 핵심 개념

**KNN (K-Nearest Neighbors)** ⭐
- 새로운 데이터가 들어오면, 가장 가까운 K개의 이웃을 찾아 다수결로 분류
- 💡 **비유**: 전학 간 첫날, 나와 성향이 비슷한 이웃 K명에게 "너희 어느 동아리야?" 물어보는 것!
- 가장 직관적이고 간단한 분류 알고리즘

**KNN 장단점**
- 장점: 원리가 간단, 구현 쉬움
- 단점: 데이터가 많으면 모든 거리를 계산해야 해서 느려짐

**K 값 선택의 중요성**
- K가 너무 작으면: 노이즈에 민감 (과적합)
- K가 너무 크면: 경계가 모호해짐 (과소적합)
- 교차 검증으로 최적 K값을 찾아야 함!

### 실제 활용
- 붓꽃(Iris) 분류: 꽃받침/꽃잎 길이로 품종 예측
- 농구 선수 분류: 스탯 데이터로 선수 출신 여부 판별
- 고객 분류: 구매 패턴으로 우수/일반 고객 분류

### 핵심 코드

```python
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score

# KNN 모델 (이웃 7개, 균일 가중치)
clf = KNeighborsClassifier(n_neighbors=7, weights='uniform')
clf.fit(X_train, y_train)

# 예측 및 정확도
y_predict = clf.predict(X_test)
print(f'정확도: {accuracy_score(y_test, y_predict):.2f}')
```

## Part 3. 혼동 행렬 & 평가 지표 (10분)

### 🤔 왜 배우는가?

단순히 "정확도 95%"라고 하면 모델이 좋은 걸까? **어떻게 틀렸는지**를 알아야 모델을 개선할 수 있어!

### 핵심 개념

**혼동 행렬 (Confusion Matrix)** ⭐
- 💡 **비유**: 거짓말 탐지기와 선생님!
  - **TP (True Positive)**: 숙제 했는데 선생님이 "해왔네!" 정확히 맞힘
  - **TN (True Negative)**: 숙제 안 했는데 선생님이 "안 했지!" 정확히 맞힘
  - **FP (False Positive)**: 숙제 안 했는데 선생님이 "해왔구나!" 잘못 칭찬
  - **FN (False Negative)**: 숙제 했는데 선생님이 "안 했지!" 오해함

**정밀도 (Precision) vs 재현율 (Recall)** ⭐⭐ (교수님 시험 출제 예고!)

| | 정밀도 (Precision) | 재현율 (Recall) |
|---|---|---|
| **뜻** | "맞다"고 예측한 것 중 진짜 맞은 비율 | 진짜 맞는 것 중 찾아낸 비율 |
| **비유** | 🎯 스나이퍼 작전 | 🕸️ 그물망 작전 |
| **중요한 경우** | 민간인 오폭 방지 (FP 줄이기) | 유해 영상 차단 (FN 줄이기) |
| **예시** | 군사 드론 폭격 | 포르노 영상 필터링 |

- 💡 **트레이드오프**: 정밀도를 올리면 재현율이 떨어지고, 재현율을 올리면 정밀도가 떨어짐!
- **F1 Score**: 둘을 모두 반영한 종합 지표

**마스크 (불리언 인덱싱)** ⭐
- 💡 **비유**: 조건에 맞는 사람만 걸러내는 마법의 필터!
- 전교생 중 파란색 옷 입은 사람만 보여주는 마법 안경
- 교수님: "수백만 번 얘기했다고 할 정도로 중요한 핵심 개념!"

### 실제 활용

**정밀도가 중요한 경우:**
- 스팸 메일 분류: 정상 메일을 스팸으로 잘못 분류하면 큰 문제!
- 군사 타격: 민간인 오폭은 절대 안 됨

**재현율이 중요한 경우:**
- 암 진단: 암 환자를 놓치면 치명적!
- 유해 콘텐츠 필터링: 나쁜 영상을 한 건이라도 놓치면 기업이 망할 수 있음

### 핵심 코드

```python
from sklearn.metrics import confusion_matrix, precision_score, recall_score, f1_score
import seaborn as sns
import matplotlib.pyplot as plt

# 혼동 행렬
cm = confusion_matrix(y_test, y_pred)
plt.figure(figsize=(5, 4))
sns.heatmap(cm, annot=True, fmt='d')
plt.show()

# 평가 지표
print(f'정밀도: {precision_score(y_test, y_pred):.2f}')
print(f'재현율: {recall_score(y_test, y_pred):.2f}')
print(f'F1 Score: {f1_score(y_test, y_pred):.2f}')

# ⭐ 마스크 (불리언 인덱싱)
mask = (y_train == 1)  # 조건에 맞는 데이터만 추출
positive_data = X_train[mask]
```

## Part 4. 실전 활용 — 데이터 분석가 관점 (3분)

### 💡 데이터 분석가가 가장 먼저 흡수해야 할 것

1. **머신러닝 7단계 패러다임** → 모든 프로젝트의 기본 뼈대
2. **Train/Test 분리** → 진짜 실력 평가의 핵심
3. **혼동 행렬 + 정밀도/재현율** → 모델 성능 진단 도구
4. **마스크 필터링** → 데이터 전처리의 필수 기술

### 활용 예시

| 개념 | 활용 |
|---|---|
| KNN | 고객 분류, 이미지 분류, 추천 시스템 |
| 혼동 행렬 | 모델 오류 패턴 분석 |
| 정밀도 | 스팸 분류, 의료 진단 (오탐 방지) |
| 재현율 | 유해 콘텐츠 차단, 암 진단 (놓치지 않기) |

## ❓ 예상 Q&A

**Q1. Train/Test를 왜 분리해야 하나?**
→ 문제집 답을 달달 외워서 100점 맞는 것과 같아. 새로운 데이터에도 잘 작동하는 진짜 실력을 키우려면 반드시 분리해야 해!

**Q2. KNN에서 K값을 잘못 선택하면?**
→ K가 너무 작으면 노이즈에 민감(과적합), 너무 크면 경계가 모호(과소적합). 교차 검증으로 최적 K를 찾아야 해!

**Q3. 정밀도와 재현율 중 뭘 봐야 하나?**
→ 비즈니스 상황에 따라 다름! 오탐(FP)이 위험하면 정밀도, 놓침(FN)이 위험하면 재현율!

**Q4. 마스크(불리언 인덱싱)가 왜 중요한가?**
→ 수백만 번 강조할 정도로 데이터 추출의 핵심! 조건에 맞는 데이터만 쏙 뽑아내는 필수 기술!
