## 📖 Part 1. 딥러닝 기초 — 9시 수업

### 1-1. 딥러닝이 뭔데? — 떡볶이 로봇 🤖

> 먼저 질문 하나 할게. **"왜 딥러닝을 배워야 할까?"**
>
> → 컴퓨터한테 "이 사진이 고양이야, 강아지야?"를 구분하게 만들고 싶으니까!
> → 그런데 컴퓨터는 숫자밖에 몰라. 그래서 **숫자로 된 판단 기준**이 필요해.

**자, 여기서 비유 하나 들게!**

> 딥러닝의 원리를 **'세상에서 가장 맛있는 떡볶이를 만드는 꼬마 요리사 로봇'**에 비유해볼게.

| 딥러닝 개념 | 떡볶이 로봇 비유 |
|------------|-----------------|
| **퍼셉트론** | 꼬마 요리사 로봇 그 자체 |
| **입력 데이터** | 떡, 고추장, 설탕 같은 재료 |
| **가중치** | 재료를 넣는 비율 (고추장 2숟갈? 설탕 반 숟갈?) |
| **오차** | 손님이 기대한 맛과 로봇이 만든 맛의 차이 |
| **학습** | 손님 평가 듣고 레시피 계속 수정하는 과정 |

> 쉽지? 💡
> 결국 딥러닝이란, 수많은 꼬마 요리사 로봇(퍼셉트론)들이 모여서
> 맛의 차이(오차)를 줄이기 위해 스스로 재료의 비율(가중치)을 계속 수정해 나가며,
> 마침내 완벽한 요리(정확한 예측)를 만들어내는 거야!
>
> 그니까 **가중치를 잘 조절하는 것 = 학습(Learning)** 이란 말이셈 ㅇㅇ

### 1-2. 퍼셉트론: 딥러닝의 가장 작은 단위

> 그럼 이 꼬마 요리사, 즉 **퍼셉트론**이 어떻게 동작하는지 좀 더 자세히 볼게.

```
입력(X1, X2, X3) × 가중치(W1, W2, W3) → 합산 → + 편향(b) → 활성화 함수 → 출력
```

> 비유하자면, **시험 점수 계산**이랑 같아 💡
> - 국어 점수 × 국어 반영 비율 + 영어 점수 × 영어 반영 비율 + ...
> - 여기서 **반영 비율이 가중치(Weight)**야!
> - 중요한 과목일수록 비율을 크게 주는 거랑, 중요한 입력일수록 가중치를 크게 주는 거랑 같아

**핵심 포인트:**
- 가중치(Weight)를 잘 조절하는 것 = **학습(Learning)** 그 자체!
- 예측값이 정답에 가까워지도록 가중치를 계속 수정하는 과정

### 1-3. 오차와 손실 함수: "얼마나 틀렸나?" 재는 법

> 근데 가중치를 어떻게 수정해야 하는지 어떻게 알아?
> → 먼저 **"얼마나 틀렸는지"**를 숫자로 알아야 하잖아!
> → 손님이 "너무 달아!" 하면 설탕을 줄여야지, "너무 매워!" 하면 고추장을 줄여야지

**MSE (Mean Squared Error) — 평균 제곱 오차:**
```
MSE = (1/N) × Σ(실제값 - 예측값)²
```

> 비유하자면, **다트 게임**이야 🎯
> - 과녁 중앙이 정답, 내가 던진 다트가 예측값
> - 중심에서 멀어질수록 오차가 커지고
> - 오차를 제곱하는 이유? → 많이 틀린 걸 더 크게 벌주려고! (방향 상관없이 크기만 재려고)

### 1-4. 경사 하강법: 오차를 줄이는 방법

> 오차를 줄이려면 가중치를 어느 방향으로 바꿔야 할까?
> → 이때 쓰는 게 **경사 하강법(Gradient Descent)**이야.

> 비유하자면, **안개 속 산 내려가기**야 🏔️
> - 눈에 안 보이니까 발로 바닥을 더듬어서 가장 경사가 급한 방향으로 한 발씩 내려가는 거야
> - 이 "경사"를 수학에서는 **미분(기울기)**이라고 불러
> - 기울기가 0이 되는 지점 = 오차가 최소인 지점 = 최적의 가중치!

```
오차 계산 → 미분으로 방향 파악 → 가중치를 조금 수정 → 다시 오차 계산 → 반복!
```

### 1-5. 활성화 함수: 비선형성의 힘

> 근데 여기서 문제가 있어. 그냥 숫자 계산만 반복하면 **직선**밖에 못 그려.
> 현실 세계는 직선으로 설명할 수 없는 복잡한 패턴이 많잖아!
> 그래서 필요한 게 **활성화 함수(Activation Function)**야.

> 비유하자면, **카메라 필터**야 📷
> - 같은 장면도 필터를 바꾸면 완전히 다른 느낌이 나잖아
> - 활성화 함수도 비슷해. 같은 숫자도 함수를 통과시키면 다른 결과가 나와

**주요 활성화 함수 3가지:**

| 함수 | 특징 | 쓰이는 곳 |
|------|------|----------|
| **Sigmoid** | 출력을 0~1 사이로 변환. "확률"처럼 표현 | 이진 분류 출력층 |
| **ReLU** | 음수는 0, 양수는 그대로. 간단하고 강력! | 은닉층 (가장 많이 사용) |
| **Softmax** | 여러 값을 확률 분포로 변환 (합계=1) | 다중 분류 출력층 |

> 💡 **Sigmoid의 문제점**: 기울기 소실(Vanishing Gradient)
> - 값이 너무 크거나 작으면 기울기가 거의 0이 되어서 학습이 멈춰버려
> - 그래서 등장한 게 **ReLU**! 양수면 기울기가 항상 1이라서 학습이 잘 돼

### 1-6. 순전파 & 역전파: 학습의 핵심 메커니즘

> 자, 이제 딥러닝이 어떻게 학습하는지 전체 흐름을 볼게.

**순전파 (Forward Propagation):**
```
입력 → 계층1 → 계층2 → ... → 출력(예측값)
```
> 데이터를 앞으로 밀어서 예측값을 만드는 과정

**역전파 (Backpropagation):**
```
오차 계산 → 출력층 → 계층2 → 계층1 → 입력 방향으로 기울기 전파 → 가중치 수정
```
> 오차를 거꾸로 돌려보내서 각 가중치를 얼마나 바꿔야 하는지 계산하는 과정

> 비유하자면, **축구 경기 영상 리뷰**야 ⚽
> - 순전파: 경기 진행 (공격 → 슛 → 골 or 실패)
> - 역전파: 골키퍼가 골을 먹혔을 때 "내가 왼쪽으로 10cm 더 움직여야 했어!" 라고 복기
> - 이 복기를 통해 다음에는 더 잘 막을 수 있게 되는 거지!

### 1-7. PyTorch로 딥러닝 모델 만들기

> 이제 실제 코드로 어떻게 구현하는지 볼게!

**모델 정의 3단계:**
```python
# 1단계: nn.Module 상속
class MyModel(nn.Module):
    
    # 2단계: __init__에서 계층 정의
    def __init__(self):
        super().__init__()
        self.linear1 = nn.Linear(3, 2)  # 입력 3개 → 출력 2개
        self.relu = nn.ReLU()
        self.linear2 = nn.Linear(2, 1)  # 입력 2개 → 출력 1개
    
    # 3단계: forward에서 데이터 흐름 정의
    def forward(self, x):
        x = self.linear1(x)
        x = self.relu(x)
        x = self.linear2(x)
        return x
```

> ⚠️ **주의할 점**: 이전 계층의 `out_features`와 다음 계층의 `in_features`는 반드시 일치해야 해!
> → 소켓이랑 플러그 맞추는 것처럼, 크기가 안 맞으면 안 들어가 ㅇㅇ

**학습 루프 5단계:**
```python
for epoch in range(epochs):
    # 1. 순전파: 예측값 만들기
    pred = model(inputs)
    
    # 2. 손실 계산: 얼마나 틀렸나?
    loss = loss_fn(pred, targets)
    
    # 3. 기울기 초기화: 이전 기울기 지우기
    optimizer.zero_grad()
    
    # 4. 역전파: 기울기 계산
    loss.backward()
    
    # 5. 가중치 업데이트: 실제로 수정!
    optimizer.step()
```

> 💡 **optimizer.step()**이 진짜로 가중치를 바꾸는 순간!
> 이때 **학습률(Learning Rate)**이 적용되어 "얼마나 크게 수정할지" 결정해

## 📖 Part 2. CNN — 이미지 인식의 핵심 (10시~11시 수업)

### 2-1. 왜 CNN이 필요할까? 🤔

> 자, 여기서 질문! **"왜 딥러닝 모델이 이미지를 인식하기 어려울까?"**
>
> 기존 Linear Layer는 마치 **사진을 통째로 외우는 '고정식 스캐너'**와 같아.

**문제 1: 위치 의존성**
> - 증명사진처럼 물체가 늘 가운데에 고정되어 있으면 잘 맞추지만
> - 일상 사진에서 강아지가 구석으로 조금만 이동하면 픽셀 위치가 달라져 **완전히 다른 사진이라고 착각해버려!**

**문제 2: 파라미터 폭발 (차원의 저주)**
> - 2차원 사진을 1차원(한 줄)으로 쭉 펴서 억지로 처리해 😱
> - 500×500 픽셀 컬러 이미지 → 250,000개 픽셀
> - 은닉층 1,000개와 연결하면? **2억 5천만 개의 가중치 필요!**
> - 중학생에게 갑자기 2억 5천만 개의 수학 문제를 풀라고 하는 것과 같아!

> 이처럼 **계산량은 줄이면서, 사진 속 물체의 위치가 바뀌어도 똑똑하게 인식하기 위해**
> **인간의 시각 구조를 흉내 내어 만든 것이 바로 CNN**이야! 🎉

### 2-2. CNN의 핵심 아이디어: 꼬마 탐정과 돋보기 🔍

> CNN은 사진을 무식하게 한 번에 외우지 않아.
> 대신 **'마법의 돋보기(필터)'를 들고 사진 구석구석을 차근차근 훑어보는 꼬마 탐정**처럼 작동해!

**CNN의 구조:**
```
입력 이미지 → [Convolution + Pooling 반복] → Feature Maps → Fully Connected → 분류 결과
              ↑_________________________↑
                  Feature Extractor (특징 추출기)
```

**특징 추출 과정 (3단계):**

| 단계 | 탐정 비유 | 실제 CNN |
|------|----------|---------|
| **Low-Level** | 단순한 돋보기로 가로선, 세로선, 색깔 같은 단순한 힌트 찾기 | 엣지, 색상 등 기본 특징 |
| **Mid-Level** | 선들을 조합한 동그라미, 곡선, 모양 알아채기 | 텍스처, 패턴 등 중간 특징 |
| **High-Level** | 자동차의 바퀴나 창문, 강아지의 눈이나 코 같은 결정적 단서 찾기 | 객체의 구체적인 부위 |

> 탐정은 찾아낸 여러 단서 조각들(바퀴, 창문, 차체 모양 등)을 모아서
> **"이 사진은 자동차네!"**라고 최종 판결(분류)을 내려!

> 💡 **CNN의 가장 놀라운 점**: 바퀴나 눈을 찾기 위해 **돋보기의 도수나 색깔(필터의 가중치 값)을
> 어떻게 설정해야 하는지 AI가 학습을 통해 '스스로' 알아낸다는 것!**
> → 사람이 일일이 공식을 알려주지 않아도, 탐정이 스스로 수많은 사건을 해결해 보며
> 자신만의 완벽한 돋보기를 만들어내는 것과 같아!

### 2-3. CNN 핵심 구성 요소 5가지

> 자, 이제 CNN을 이루는 핵심 부품들을 하나씩 살펴볼게!

---

#### ① 필터(Filter/Kernel) — 특수 안경 🥽

> **비유**: 특정 색깔이나 질감만 두드러지게 보여주는 **'특수 안경'**이나 **'카메라 렌즈'**

**역할:**
- 이미지 위를 돌아다니며 선, 모서리, 질감 같은 **특정 패턴(특징)만 쏙쏙 뽑아내어**
  새로운 **'특징 맵(Feature Map)'**을 만들어냄
- AI가 이미지 안의 중요한 특징들을 스스로 학습하고 포착할 수 있게 해주는 **가장 필수적인 도구!**

**컨볼루션 연산 과정:**
```
1. 필터(예: 3×3)를 이미지 위에 올려놓기
2. 필터와 겹치는 픽셀들을 각각 곱하기 (Element-wise Multiplication)
3. 곱한 값들을 모두 더하기 → 하나의 값!
4. 필터를 한 칸씩 이동(Sliding)하며 반복
5. 결과: 새로운 2D 배열 = Feature Map (특징 맵)
```

> 💡 필터의 값은 AI가 **학습을 통해 스스로** 만들어! 사람이 직접 설계하는 게 아니야.

---

#### ② 패딩(Padding) — 보호용 빈 액자 틀 🖼️

> **비유**: 사진 테두리가 잘려 나가지 않게 덧대어주는 **'보호용 빈 액자 틀'**

**역할:**
- 필터를 사용해 이미지를 훑다 보면 결과물의 크기가 원본보다 계속 작아져 😢
- 6×6 이미지에 3×3 필터를 적용하면 → 4×4로 줄어!
- 원본 이미지 바깥 테두리에 0이라는 빈 공간을 채워 넣어 **이미지 크기를 원래대로 유지**시킴

**두 가지 목적:**
1. **Feature Map 크기 유지**: 컨볼루션으로 인한 크기 축소 방지
2. **모서리 특징 보존**: 패딩 없이 필터로 사진을 훑으면 정중앙은 여러 번 훑지만
   모서리(가장자리)는 한두 번만 훑게 되어 모서리의 중요한 정보를 놓치게 돼.
   여백을 주면 **모서리 픽셀도 여러 번 연산에 포함**되어 테두리 주변의 특징을 강화할 수 있어!

> PyTorch에서는 `padding='same'`을 사용하면 자동으로 크기 맞춰줘 ㅇㅇ

---

#### ③ 스트라이드(Stride) — 탐정의 발걸음 보폭 🚶

> **비유**: 돋보기를 들고 단서를 찾을 때 걷는 **'발걸음 보폭'**

**역할:**
- 필터가 이미지 위를 스캔하며 이동할 때 **몇 칸씩 건너뛰며 이동할지 결정하는 간격**

| 보폭 | 효과 |
|------|------|
| **Stride 1** | 한 칸씩 이동 → 디테일하게 특징 추출, 출력 크기 큼, 계산 느림 |
| **Stride 2** | 두 칸씩 이동 → 빠르게 훑음, 출력 크기 작아짐, 계산 빠름 |

> 비유하자면, **책 읽기**야 📖
> - Stride 1: 한 줄씩 꼼꼼히 읽기
> - Stride 2: 두 줄씩 건너뛰며 빨리 읽기 (요약본 느낌)

> ⚠️ **주의**: Stride가 클수록 출력 Feature Map의 크기가 **더 작아져**!

---

#### ④ 풀링(Pooling) — 반장 선거 🗳️

> **비유**: 구역별로 가장 뛰어난 대표 한 명만 뽑아 인원을 줄이는 **'반장 선거'** 또는 **'핵심 요약'**

**역할:**
- 특정 영역 안에서 가장 큰 값(Max Pooling)이나 평균값(Average Pooling) 등
  **대표값만 추출하여 전체 데이터 크기를 절반 정도로 확 줄이는(다운샘플링) 과정**
- 계산량을 획기적으로 감소시킴
- 사진 속 강아지가 왼쪽이나 오른쪽으로 약간 이동하더라도(위치 변화)
  여전히 강아지로 인식할 수 있게끔 **견고함(위치 불변성)을 부여**!

```
Max Pooling 예시:
[1, 3]    → [3]     ← 2×2 영역에서 최댓값만 추출!
[2, 4]    

→ 가로/세로가 절반으로 줄어듦!
```

---

#### ⑤ Global Average Pooling(GAP) — 만능 압축기 📦

> **비유**: 크기가 제각각인 서류 뭉치가 들어와도 무조건 한 장짜리 규격 보고서로 만들어주는 **'만능 압축기'**

**왜 필요할까?**
> 일반적인 CNN 모델의 마지막 Linear 계층은 입력받을 데이터의 크기(`in_features`)가 **정확하게 고정**되어 있어야 작동해.

> 만약 32×32 이미지 기준으로 `in_features`를 계산해 놓았는데,
> **입력 이미지가 64×64로 바뀌면** → 크기가 맞지 않아 **에러 발생!** 모델이 멈춰버려! 😱

**GAP의 해결책:**
- 입력 이미지의 크기가 변하더라도 **모델이 피처 크기를 동적으로 알아서 조절**해 줌
- 다양한 해상도의 이미지가 들어와도 에러 없이 유연하게 분류기를 통과할 수 있어!
- 현대 CNN 모델에서 **필수적으로 사용**되는 기법이야

## 📖 Part 3. PyTorch CNN 코드 실습 (14시~15시 수업)

### 3-1. CNN 모델 구현하기

> 이제 실제로 CNN 모델을 코드로 만들어볼게!

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN_01(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        
        # 특징 추출 계층
        self.conv_1 = nn.Conv2d(in_channels=3, out_channels=32, kernel_size=3)
        self.conv_2 = nn.Conv2d(in_channels=32, out_channels=64, kernel_size=3)
        self.pool = nn.MaxPool2d(kernel_size=2)
        
        # 분류 계층
        self.flatten = nn.Flatten()
        self.classifier = nn.Linear(in_features=64 * 14 * 14, out_features=num_classes)

    def forward(self, x):
        # Conv → ReLU → Conv → ReLU → Pool → Flatten → Linear
        x = F.relu(self.conv_1(x))
        x = F.relu(self.conv_2(x))
        x = self.pool(x)
        x = self.flatten(x)
        x = self.classifier(x)
        return x
```

**Feature Map 크기 변화 추적 (28×28 입력 기준):**
```
입력:        [3, 28, 28]
Conv1 (k=3): [32, 26, 26]    ← (28-3)/1+1 = 26
Conv2 (k=3): [64, 24, 24]    ← (26-3)/1+1 = 24
Pool (k=2):  [64, 12, 12]    ← 24/2 = 12
Flatten:     [9216]           ← 64 × 12 × 12
Linear:      [10]             ← 10개 클래스로 분류
```

### 3-2. GAP 적용 모델

```python
class SimpleCNN_02(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.conv_1 = nn.Conv2d(3, 32, kernel_size=3)
        self.conv_2 = nn.Conv2d(32, 64, kernel_size=3)
        self.pool = nn.MaxPool2d(kernel_size=2)
        
        # ✨ Flatten 대신 GAP 사용!
        self.adapt_pool = nn.AdaptiveAvgPool2d(output_size=(1, 1))
        
        # in_features가 채널 수(64)로 고정!
        self.classifier = nn.Linear(in_features=64, out_features=num_classes)

    def forward(self, x):
        x = F.relu(self.conv_1(x))
        x = self.pool(x)
        x = F.relu(self.conv_2(x))
        x = self.pool(x)
        
        x = self.adapt_pool(x)            # [64, 12, 12] → [64, 1, 1]
        x = torch.flatten(x, start_dim=1) # [64, 1, 1] → [64]
        x = self.classifier(x)            # [64] → [10]
        return x
```

**Flatten vs GAP 비교:**

| | Flatten | GAP |
|--|---------|-----|
| 방식 | 모든 값을 1D로 펼침 (9216개) | 각 채널의 평균값 1개만 추출 (64개) |
| 파라미터 | 많음 → 오버피팅 위험 | 적음 → 오버피팅 방지 |
| 입력 크기 | 고정 필요 (32×32만 가능) | 유연함! (32×32든 64×64든 OK) |
| 비유 | 모든 서류를 일렬로 늘어놓기 | 서류 뭉치를 한 장 요약본으로 압축 |

> 💡 **GAP의 핵심**: 입력 이미지 크기가 32×32든 64×64든,
> `AdaptiveAvgPool2d`가 항상 1×1로 압축해주니까
> Linear 레이어의 in_features는 **항상 64로 고정!**
> → **입력 크기에 유연한 모델**이 되는 거야! 🎉

## 📖 Part 4. 16시 수업 — 전체 정리 & 실전 완성

### 4-1. 오늘 하루 전체 복습 📝

> 자, 오늘 7교시 동안 엄청나게 많은 걸 배웠어. 전체 한 번 정리해볼게!

**전체 학습 워크플로우:**
```
📊 데이터 준비
   ↓
🏗️ 모델 설계 (nn.Module 상속, 계층 정의)
   ↓
⚙️ 손실 함수 & 옵티마이저 설정
   ↓
🔄 학습 루프 (순전파 → 손실 계산 → 역전파 → 가중치 업데이트)
   ↓
📈 검증 & 평가
   ↓
🔧 하이퍼파라미터 튜닝
```

**CNN 모델 설계 핵심 요약:**
```
입력 이미지
    ↓
[Conv2d → ReLU → Pooling] × N  ← Feature Extraction (특징 추출)
    ↓
Flatten 또는 GAP               ← 3D → 1D 변환
    ↓
Fully Connected (Linear)        ← Classification (분류)
    ↓
Softmax                         ← 확률 출력
    ↓
최종 분류 결과!
```

### 4-2. 교수님 강조 포인트 총정리 ⭐

> 오늘 수업에서 교수님이 특히 강조하신 부분 정리해줄게!

1. **가중치(Weight) = 학습(Learning) 그 자체!**
   - 가중치를 잘 맞춰나가는 게 곧 학습이야
   - 복잡한 수학 공식보다 이 개념적 이해가 더 중요해 ㅇㅇ

2. **Feature Map 크기 계산 공식 필수!**
   - `(입력크기 - 커널크기 + 2 × 패딩) / 스트라이드 + 1`
   - 이거 모르면 모델 설계 못 하니까 외워둬!

3. **다채널 컨볼루션 원리**
   - 입력 채널 수(예: RGB 3개) = 필터의 채널 수
   - 출력 채널 수 = 필터의 개수
   - 이 두 개념 헷갈리지 마 ㅗㅗ

4. **GAP는 이제 필수!**
   - 현대 CNN 모델에서는 Flatten 대신 GAP를 기본으로 써
   - 입력 크기 유연 + 파라미터 절감 + 오버피팅 방지 일석삼조!

5. **계층이 깊어질수록 채널 수는 ↑, 공간 크기는 ↓**
   - 초반: 단순한 특징(엣지, 색상) → 적은 채널
   - 후반: 복잡한 특징(물체 부위, 질감) → 많은 채널
   - 이 패턴 이해하면 CNN 구조 설계가 쉬워져!

### 4-3. 실전 팁 💡

> 실제로 모델 만들 때 유용한 팁 몇 가지 알려줄게!

- **`padding='same'`**: 크기 유지하고 싶을 때 쓰면 됨, PyTorch가 자동 계산해줘
- **`nn.AdaptiveAvgPool2d((1,1))`**: GAP의 표준 구현, 어떤 입력 크기든 OK
- **`torch.flatten(x, start_dim=1)`**: 배치 차원은 유지하고 나머지만 펼칠 때 사용
- **`summary()` 함수**: 모델 구조 한눈에 보고 싶을 때 `torchinfo` 라이브러리 사용하기
