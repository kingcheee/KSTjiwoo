## Part 1. 과대적합 & 과소적합 (8분)

### 🤔 왜 배우는가?

모델을 만들었는데 훈련 점수는 99%인데 테스트 점수는 빵점? **왜 이런 일이 발생하는지**를 알아야 모델을 고칠 수 있어!

### 핵심 개념

**과대적합 (Overfitting)** ⭐⭐ (교수님: "시험 문제 꼭 낼 거예요!")
- 💡 **비유**: 문제집을 통째로 달달 외운 학생!
- 훈련 데이터 점수 99% → 테스트 데이터 점수 빵점 (심지어 -143점!)
- 원인: 모델이 너무 복잡하거나, 훈련 데이터가 너무 적음
- 교수님: "빵부! 혹시 자기 얘기하는 거 같다고 생각하는 분 계신가요?"

**과소적합 (Underfitting)**
- 💡 **비유**: 아예 공부를 안 한 학생!
- 훈련 점수도 낮고, 테스트 점수도 낮음
- 원인: 모델이 너무 단순하거나 학습이 부족

**다항 특성 (PolynomialFeatures)**
- 기존 특성을 곱하거나 제곱해서 비선형 데이터를 학습할 수 있게 함
- ⚠️ 차수(Degree)를 너무 높이면 과대적합의 주범!

### 진단 방법
- 훈련 점수 vs 테스트 점수 그래프를 그려서 확인
- 격차가 크면 과대적합, 둘 다 낮으면 과소적합

### 실제 활용
- 모델 성능이 안 나올 때 가장 먼저 확인해야 할 것!
- 훈련/테스트 점수 그래프 → 과대적합 판정 → 규제 적용

### ⭐ 교수님 강조 포인트
> **"이거 너무 중요해요. 과대적합, 과소적합은 여러분 제가 시험 문제 꼭 낼 거예요. 틀리게 말해 봐!"**

> "훈련 데이터에는 99.9%가 나왔는데 테스트에서는 -143이 나왔어요. 이게 과대적합이에요!"

## Part 2. 규제 (Regularization) (8분)

### 🤔 왜 배우는가?

과대적합을 해결하는 가장 강력한 도구! **모델의 학습을 일부러 방해해서** 새로운 데이터에도 잘 작동하게 만들어!

### 핵심 개념

**규제 (Regularization)** ⭐
- 💡 **비유**: 선생님의 잔소리와 공부법 교정!
- "핵심만 공부해!"라고 모델의 자유도를 제한
- 알파(α) 값으로 규제 강도 조절

**Ridge (릿지) vs Lasso (라쏘)**
- **Ridge**: 모든 특성을 다 사용하되, 중요하지 않은 특성의 영향력을 0에 가깝게 줄임
- **Lasso**: 불필요한 특성은 아예 제거(계수를 정확히 0으로)해서 핵심 특성만 선택

**알파(α) 값 조절** ⭐
- α를 **크게** 하면 → 모델이 단순해짐 → 과소적합 위험
- α를 **작게** 하면 → 모델이 복잡해짐 → 과대적합 위험
- **최적의 α**: 훈련 점수와 테스트 점수의 격차가 가장 작으면서 테스트 점수가 높은 지점!

### 핵심 코드

```python
from sklearn.linear_model import Ridge, Lasso
from sklearn.preprocessing import PolynomialFeatures

# 다항 특성 생성 (차수 조절 주의!)
poly = PolynomialFeatures(degree=2, include_bias=False)
X_train_poly = poly.fit_transform(X_train)

# ⭐ Ridge 규제 (alpha 값으로 강도 조절)
ridge = Ridge(alpha=1.0)
ridge.fit(X_train_poly, y_train)

# ⭐ Lasso 규제 (불필요한 특성 제거)
lasso = Lasso(alpha=0.1)
lasso.fit(X_train_poly, y_train)

# 최적 alpha 찾기
from sklearn.model_selection import GridSearchCV
params = {'alpha': [0.001, 0.01, 0.1, 1, 10]}
grid = GridSearchCV(Ridge(), params, cv=5)
grid.fit(X_train_poly, y_train)
print(f'최적 alpha: {grid.best_params_}')
```

### 실제 활용
- 모델 성능이 안 나올 때 가장 먼저 시도하는 방법
- 알파 값을 바꿔가며 그래프를 그려서 최적값 찾기
- 딥러닝에서도 Dropout, Weight Decay 등 규제 기법의 기초

### ⭐ 교수님 강조 포인트
> "규제를 쓰는 거야. 모델을 좀 더 단순화시킨다는 거예요!"

## Part 3. 결정 트리 (Decision Tree) (7분)

### 🤔 왜 배우는가?

딥러닝은 "왜 그런 결과가 나왔는지" 설명 못 해. 결정트리는 **설명 가능한 AI**를 만들 수 있어!

### 핵심 개념

**결정 트리** ⭐
- 💡 **비유**: 스무고개 게임 (아키네이터)!
- "기온이 30도 이상이야?" → "바람이 2m/s 이상이야?" 처럼 질문을 던져서 분류
- **화이트 박스(White-box) 모델**: 왜 그런 결과가 나왔는지 과정을 명확하게 보여줌!

**엔트로피 (Entropy)** ⭐
- 💡 **비유**: 어질러진 장난감 상자의 혼잡도!
- 로봇과 인형이 반반 섞여 있으면 엔트로피 최고 (예측 불가)
- 한 종류만 있으면 엔트로피 0 (완벽한 분류)

**정보 이득 (Information Gain)**
- 💡 **비유**: 장난감 상자를 특정 기준으로 나눴을 때 얼마나 깔끔하게 정리되었는지!
- 정보 이득이 가장 큰 질문을 먼저 던져서 트리를 성장시킴

### 핵심 코드

```python
from sklearn.tree import DecisionTreeClassifier, plot_tree

# 결정 트리 (max_depth로 과적합 방지!)
clf = DecisionTreeClassifier(max_depth=3, random_state=42)
clf.fit(X_train, y_train)

# 시각화
plt.figure(figsize=(20, 10))
plot_tree(clf, feature_names=feature_names, class_names=class_names, filled=True)
plt.show()
```

### 실제 활용
- **대출 심사**: "소득이 3천만원 이하이고 연체 기록이 있어서 거절"처럼 이유 설명 가능
- **고객 이탈 예측**: 어떤 조건에서 고객이 이탈하는지 명확히 파악
- **의료 진단**: 어떤 증상 조합으로 질병을 판단했는지 설명

### ⭐ 교수님 강조 포인트
> "결정트리는 화이트 박스 모델이에요. 왜 이런 결과가 나왔는지 과정을 명확하게 보여줄 수 있어요!"

> "트리 깊이(max_depth)에 제한을 두지 않으면 완벽하게 0이 될 때까지 끝없이 가지를 치게 되어 과대적합에 매우 빠지기 쉬워요!"

## Part 4. 교차 검증 & 실전 활용 (4분)

### 핵심 개념

**교차 검증 (Cross Validation)** ⭐
- 💡 **비유**: 모둠별 모의고사 돌려 풀기!
- 데이터를 5등분 → 4개로 훈련, 1개로 검증 → 5번 반복
- 적은 데이터로도 다양한 문제를 풀어본 효과!

**데이터 스케일링 (StandardScaler)** ⭐
- 💡 **비유**: 심사위원 점수를 공정하게 조정하기!
- 특성 간 값 차이가 크면 큰 값이 지배하게 됨
- ⭐ **훈련 데이터**: `fit_transform()` — 평균/편차 새로 계산
- ⭐ **테스트 데이터**: `transform()`만 — 기존 기준 적용

### ⭐ 교수님 강조 포인트
> "스케일링 시 훈련 데이터(X_train)에는 fit_transform을 사용해 평균과 편차를 새로 구해야 하지만, 테스트 데이터(X_test)에는 절대 새로 계산하지 말고 transform만 사용해야 한다. 꼭 기억하라!"
