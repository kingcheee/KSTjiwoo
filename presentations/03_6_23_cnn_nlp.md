## Part 1. CNN 심화 — 배치 정규화 & 드롭아웃 (5분)

### 🤔 왜 배우는가?

CNN으로 이미지 분류를 만들었는데, **모델이 너무 깊어지면 학습이 불안정해져**. 데이터가 한쪽으로 쏠리고, 기울기가 사라지고, 과적합이 발생해. 이걸 해결하는 기법들이야!

### 핵심 개념

**배치 정규화 (Batch Normalization)** ⭐
- 층을 지날 때마다 데이터 분포가 왜곡되는 걸 막아줌
- 교수님: **"성능이 5~10% 이상 차이남. 요즘은 무조건 깔고 가는 필수 옵션!"**
- 💡 **비유**: 엇나가는 학생을 잡아주는 **선생님의 잔소리**. "진정해, 거긴 안 나와, 다시 원래 진도로 돌아와~"

**드롭아웃 (Dropout)** ⭐
- 학습 시 랜덤하게 뉴런을 꺼서 과적합 방지
- 교수님: **"드롭아웃은 오직 훈련 시에만 적용! 테스트 시에는 절대 적용하면 안 됨!"**
- 💡 **비유**: 공부에 미친 친구의 **책 뺏고 강제로 쉬게 하기**

**nn.Sequential 모듈화** ⭐
- 복잡한 레이어를 레고 블록처럼 묶어서 관리
- **가지치기**: 특정 블록만 떼어내서 동적으로 교체 가능

**Global Average Pooling**
- 입력 이미지 크기가 달라도 출력을 1x1로 고정

### 핵심 코드

```python
import torch.nn as nn

self.conv_block = nn.Sequential(
    nn.Conv2d(in_channels=3, out_channels=32, kernel_size=3, padding='same'),
    nn.BatchNorm2d(num_features=32),
    nn.ReLU(),
    nn.MaxPool2d(kernel_size=2)
)

self.classifier_block = nn.Sequential(
    nn.AdaptiveAvgPool2d(output_size=(1, 1)),
    nn.Flatten(start_dim=1),
    nn.Dropout(p=0.5),
    nn.Linear(in_features=128, out_features=10)
)
```

## Part 2. CNN 아키텍처 복습 (3분)

**CNN 필터** = 이미지에서 특징을 추출하는 **마법의 특수 안경(렌즈)**
- AI가 학습을 통해 스스로 최적의 값을 찾아냄

**패딩** = 사진 테두리에 덧대는 **보호용 빈 액자 틀**

**풀링** = 구역마다 **반장(조별 대표) 뽑기**

**설계 공식** ⭐
- 입력 채널 수 = 필터의 채널 수
- 필터의 개수 = 출력 Feature Map의 채널 수

## Part 3. NLP 기초 — 토큰화 & 임베딩 (5분)

### 🤔 왜 배우는가?

컴퓨터는 텍스트를 이해 못 해. **텍스트를 숫자로 바꿔야** AI가 학습할 수 있어!

### 핵심 개념

**토큰화 (Tokenization)** ⭐
- 문장을 의미 있는 단위로 자르고 숫자 ID 부여
- 💡 **비유**: 통닭을 **한 입 크기로 토막 내기**

**원-핫 인코딩의 단점** ⭐
- 단어 1만 개면 9,999개의 0 → 메모리 낭비
- 단어 간 유사성 파악 불가
- 💡 **비유**: 엄청난 **메모리 낭비와 눈치 부족**

**워드 임베딩 (Word Embedding)** ⭐
- 단어를 다차원 벡터로 변환해서 의미적 유사도 계산
- 💡 **비유**: 단어들에게 의미를 부여하는 **마법의 3차원 위치 좌표**
- King-Queen ≈ Man-Woman (비슷한 관계!)

### 핵심 코드

```python
from transformers import AutoTokenizer, AutoModel

tokenizer = AutoTokenizer.from_pretrained('bert-base-uncased')
inputs = tokenizer("Hello, world!", return_tensors='pt')

embedding = nn.Embedding(
    num_embeddings=vocab_size,
    embedding_dim=32,
    padding_idx=0
)
```

## Part 4. 실전 활용 — 데이터 분석가 관점 (3분)

### 💡 가장 먼저 흡수해야 할 것

1. **배치 정규화 + 드롭아웃** → 모델 성능과 안정성의 필수 옵션
2. **토큰화 + 임베딩** → 텍스트 데이터 분석의 기초 체력
3. **nn.Sequential 모듈화** → 코드 재사용성 향상

### 활용 예시

| 개념 | 활용 |
|---|---|
| CNN | 제품 불량 탐지, 의료 영상 분석 |
| 배치 정규화 | 모든 딥러닝 모델의 필수 옵션 |
| 드롭아웃 | 실서비스 배포 시 모델 안정성 |
| 토큰화 | 스팸 분류, 감성 분석의 첫 단계 |
| 임베딩 | 사내 챗봇, 의미 기반 검색 |

## 📋 핵심 요약 (1분)

2. **토큰화 + 임베딩** = 텍스트를 숫자로 바꾸는 기초 체력
3. **nn.Sequential 모듈화** = 레고 블록처럼 코드를 묶어서 재사용!
