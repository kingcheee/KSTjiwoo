## Part 1. RNN — 순환 신경망 (4분)

### 💡 왜 배우는가?

여러분, 카톡 대화 생각해봐. "나 어제 그 카페 갔었는데 **거기** 진짜 좋더라" — 이 문장에서 **거기**가 뭘 의미하는지 알려면 앞의 맥락을 기억해야 하잖아? 컴퓨터도 이렇게 순차적인 데이터를 이해하려면 **기억**이 필요해. 그게 바로 RNN이야!

### 🔑 핵심 개념: 히든 스테이트 (Hidden State)

**쉽게 말하면: 릴레이 소설의 줄거리 요약 메모** 📝

- 단어가 하나씩 순차적으로 들어올 때, 앞서 나온 단어들의 맥락을 까먹지 않기 위해 잠시 머릿속에 담아두는 **단기 기억 공간**이야
- 앞 사람의 이야기를 듣고 내 이야기를 이어가기 위해 직전까지의 맥락을 짧게 기억하는 것과 같아

### ❌ 한계: 기울기 소실 (Gradient Vanishing)

긴 문장이나 긴 데이터가 들어오면... 앞부분을 까먹어버려! 마치 긴 글 다 쓰고 나서 첫 문단이 뭐였지? 하는 거랑 같아.

**이게 왜 문제?** → 문장 앞부분의 중요한 정보를 잃어버리면 AI가 엉뚱한 답을 하게 돼

## Part 2. LSTM — 장기 기억을 살리자! (5분)

### 💡 왜 배우는가?

RNN이 까먹는 문제를 해결하려고 등장한 게 **LSTM(Long Short-Term Memory)**이야. 데이터 분석에서 장기 트렌드를 분석하려면 이게 필수거든! 예를 들어 작년 1월부터 지금까지의 주가 흐름을 분석하려면, 1월 데이터를 기억하고 있어야 하잖아? 🤔

### 🔑 핵심 개념: 셀 스테이트 + 3개의 게이트

**쉽게 말하면: 오래된 비밀 노트 + 스마트 지우개/형광펜** 📓✨

LSTM은 두 가지 기억 공간을 가져:

1. **히든 스테이트** — 단기 기억 (지금 이 순간의 맥락)
2. **셀 스테이트** — 장기 기억 (중요한 것만 따서 보관하는 비밀 노트)

그리고 이 노트를 관리하는 **문지기(게이트)**가 3명 있어:

| 게이트 | 역할 | 비유 |
|--------|------|------|
| **망각 게이트** | 필요 없는 정보 버리기 | 지우개로 안 중요한 거 지우기 🧹 |
| **업데이트 게이트** | 새로운 중요 정보 추가 | 형광펜으로 중요한 거 칠하기 🖍️ |
| **출력 게이트** | 지금 필요한 정보만 꺼내기 | 시험 볼 때 그 부분만 펼쳐보기 📖 |

### 🎯 실생활 예시

**금융권 이상 거래 탐지** 💳
- 여러분의 카드 사용 패턴을 LSTM이 기억해
- "한국에서 결제한 지 10분 만에 미국에서 고액 결료" → 비정상 패턴 감지!
- 셀 스테이트가 "이 사람은 평소 서울에서만 결제한다"는 장기 기억을 보관하고 있어서 가능한 거야

**스마트 팩토리 예지 보전** 🏭
- 공장 센서 데이터를 1초 단위로 분석
- "과거 데이터 흐름 보니 3일 뒤 이 부품 고장 날 확률 높다" → 미리 교체!

## Part 3. 트랜스포머 — 순서 따위 버리자! (5분)

### 💡 왜 배우는가?

RNN과 LSTM의 가장 큰 문제: **데이터를 하나씩 순차적으로 처리**해서 느리다! GPU를 100개 줘도 한 번에 하나씩 처리하면 의미 없잖아? 트랜스포머는 이걸 바꿔놓었어. 지금 우리가 쓰는 ChatGPT도 트랜스포머 기반이야!

### 🔑 핵심 개념: 병렬 연산 (Parallel Processing)

**쉽게 말하면: 조별 과제의 완벽한 동시 분업** 👥

- **RNN/LSTM**: 100장짜리 책을 혼자 1쪽부터 순서대로 읽기 → 느려!
- **트랜스포머**: 친구 100명을 불러모아 각자 1장씩 **한 방에 동시에 읽고** 정보 합치기 → 빨라!

### 🔑 핵심 개념: 어텐션 (Attention)

트랜스포머는 모든 단어를 동시에 보면서 **어떤 단어가 서로 관련 있는지** 파악해. "나 어제 그 카페 갔었는데 **거기** 좋더라" → **거기**는 **카페**와 연결된다! 이걸 수학적으로 계산하는 거야.

### 🎯 실생활 예시

- **ChatGPT, 클로드** → 트랜스포머 기반 LLM
- **넷플릭스 추천** → 사용자의 취향 벡터와 비슷한 영화를 찾아줘
- **구글 검색** → "서울 날씨 어때?"와 "오늘 명동 비 오나?"가 같은 의미라는 걸 파악

### 📊 RNN vs 트랜스포머 비교

| 항목 | RNN/LSTM | 트랜스포머 |
|------|----------|-----------|
| 처리 방식 | 순차적 (하나씩) | 병렬 (동시에) |
| 속도 | 느림 | 빠름 |
| GPU 활용 | 제한적 | 극대화 |
| 현재 용도 | 시계열, 센서 데이터 | NLP, LLM (대부분의 AI) |

## Part 4. PyTorch 실습 핵심 코드 (5분)

### 💡 왜 배우는가?

이론만 알면 의미 없지! 데이터 분석가는 **코드로 구현할 수 있어야 해**. PyTorch는 파이썬으로 AI 모델을 만드는 가장 인기있는 도구야.

### 🔑 임베딩 (Embedding)

**비유: 단어들의 MBTI 및 능력치 분석** 🧠

```python
self.embedding = nn.Embedding(num_embeddings=vocab_size, embedding_dim=embedding_dim)
```

- 컴퓨터는 글자를 못 이해해 → 숫자로 바꿔줘야 해!
- 단어마다 '성별, 귀족, 나이, 음식' 같은 수많은 차원의 점수를 매겨
- '왕(King)'과 '여왕(Queen)'은 성별 점수만 다르고 귀족 점수는 똑같이 높아
- → 컴퓨터가 "이 두 단어는 비슷한 뜻이구나!" 하고 수학적으로 파악할 수 있어

### 🔑 드롭아웃 (Dropout)

**비유: 독선적인 심사위원 눈 가리기** 🎭

```python
self.dropout = nn.Dropout(p=0.5)
```

- AI가 훈련 데이터를 너무 외워버리면 새로운 데이터에서 못 풀어 (과적합!)
- 학습 중에 뉴런을 **랜덤으로 꺼버려서** 특정 뉴런에만 의존하지 못하게 해
- ⚠️ **주의: 훈련 때만! 실제 추론(시험) 때는 꼭 꺼야 해!**

### 🔑 LSTM 구현 코드

```python
self.lstm = nn.LSTM(input_size=embedding_dim, hidden_size=hidden_size, batch_first=True)
```

- `input_size`: 임베딩 차원 (단어 한 개가 몇 차원 벡터로 표현되는지)
- `hidden_size`: 히든 스테이트의 크기 (기억 공간의 용량!)
- `batch_first=True`: 데이터 형태를 (배치, 시퀀스, 피처)로 설정

### 🔑 전체 모델 구조

```python
# 1. 입력 → 임베딩: 글자(정수) → 의미 담은 실수 벡터
embedded = self.embedding(x)

# 2. 드롭아웃: 50% 뉴런 꺼서 과적합 방지
dropped = self.dropout(embedded)

# 3. LSTM 통과: 장기 기억 + 단기 기억 생성
lstm_out, (hidden, cell) = self.lstm(dropped)

# 4. 마지막 기억만 가져와서 최종 결과 도출
last_hidden = hidden[-1]
output = self.fc(last_hidden)
```

### ⚠️ 교수님 강조 포인트 (시험 출제!)

1. **Dropout은 반드시 '훈련' 시에만!** 평가/추론 시 절대 적용 ❌
2. **사전 학습 모델 + 토크나이저는 무조건 짝꿍으로!** 다른 거 쓰면 에러!
3. **임베딩 생성 시 `padding_idx=0` 필수!** 의미 없는 PAD 토큰까지 학습하면 안 돼
4. **검증/테스트 데이터는 섞지 말 것! (No Shuffle)** 훈련만 섞어!
5. **팀 프로젝트 전 Git/GitHub 완벽 숙지!** 모르면 한 달 짠 코드 다 날아가!

## Part 5. 종합 정리 & 데이터 분석 활용 (3분)

### 🎯 이 개념이 데이터 분석에서 쓰이는 곳

| 개념 | 데이터 분석 활용 |
|------|-----------------|
| **RNN** | 시계열 데이터 예측 (주가, 날씨) |
| **LSTM** | 금융 이상 탐지, IoT 센서 예지 보전 |
| **트랜스포머** | 텍스트 분석, 감성 분류, 챗봇 |
| **임베딩** | 추천 시스템, 검색 엔진, 고객 세분화 |
| **드롭아웃** | 모델 안정화, 과적합 방지 (모든 딥러닝) |

### 📝 핵심 요약 3줄

1. **RNN**은 순차 데이터의 맥락을 기억하지만, 긴 데이터에서는 까먹어!
2. **LSTM**은 셀 스테이트와 게이트로 장기 기억을 보존해 → 시계열 분석에 강력!
3. **트랜스포머**는 병렬 연산으로 속도를 극대화 → 현대 AI의 표준!
