## 📖 Part 1. 딥러닝 기초 — 9시 수업 내용

### 1-1. 퍼셉트론: 딥러닝의 가장 작은 단위

> 먼저 질문 하나 할게. **"왜 딥러닝을 배워야 할까?"**
> 
> → 컴퓨터한테 "이 사진이 고양이야, 강아지야?"를 구분하게 만들고 싶으니까!
> → 그런데 컴퓨터는 숫자밖에 몰라. 그래서 **숫자로 된 판단 기준**이 필요해.
> 
> 이 판단 기준의 가장 기본 단위가 바로 **퍼셉트론(Perceptron)**이야.

**퍼셉트론 작동 원리:**
```
입력(X1, X2, X3) × 가중치(W1, W2, W3) → 합산 → + 편향(b) → 활성화 함수 → 출력
```

> 쉽게 비유하자면, **시험 점수 계산**이랑 같아 💡
> - 국어 점수 × 국어 반영 비율 + 영어 점수 × 영어 반영 비율 + ...
> - 여기서 **반영 비율이 가중치(Weight)**야!
> - 중요한 과목일수록 비율을 크게 주는 거랑, 중요한 입력일수록 가중치를 크게 주는 거랑 같아

**핵심 포인트:**
- 가중치(Weight)를 잘 조절하는 것 = **학습(Learning)** 그 자체!
- 예측값이 정답에 가까워지도록 가중치를 계속 수정하는 과정

### 1-2. 오차와 손실 함수: "얼마나 틀렸나?" 재는 법

> 근데 가중치를 어떻게 수정해야 하는지 어떻게 알아?
> → 먼저 **"얼마나 틀렸는지"**를 숫자로 알아야 하잖아!
> 
> 이때 쓰는 게 **오차(손실 함수, Loss Function)**야.

**MSE (Mean Squared Error) — 평균 제곱 오차:**
```
MSE = (1/N) × Σ(실제값 - 예측값)²
```

> 비유하자면, **다트 게임**이야 🎯
> - 과녁 중앙이 정답, 내가 던진 다트가 예측값
> - 중심에서 멀어질수록 오차가 커지고
> - 오차를 제곱하는 이유? → 많이 틀린 걸 더 크게 벌주려고! (방향 상관없이 크기만 재려고)

### 1-3. 경사 하강법: 오차를 줄이는 방법

> 오차를 줄이려면 가중치를 어느 방향으로 바꿔야 할까?
> → 이때 쓰는 게 **경사 하강법(Gradient Descent)**이야.

> 비유하자면, **안개 속 산 내려가기**야 🏔️
> - 눈에 안 보이니까 발로 바닥을 더듬어서 가장 경사가 급한 방향으로 한 발씩 내려가는 거야
> - 이 "경사"를 수학에서는 **미분(기울기)**이라고 불러
> - 기울기가 0이 되는 지점 = 오차가 최소인 지점 = 최적의 가중치!

**핵심 흐름:**
```
오차 계산 → 미분으로 방향 파악 → 가중치를 조금 수정 → 다시 오차 계산 → 반복!
```

### 1-4. 활성화 함수: 비선형성의 힘

> 근데 여기서 문제가 있어. 그냥 숫자 계산만 반복하면 **직선**밖에 못 그려.
> 현실 세계는 직선으로 설명할 수 없는 복잡한 패턴이 많잖아!
> 
> 그래서 필요한 게 **활성화 함수(Activation Function)**야.

> 비유하자면, **필름 카메라의 필터**야 📷
> - 같은 장면도 필터를 바꾸면 완전히 다른 느낌이 나잖아
> - 활성화 함수도 비슷해. 같은 숫자도 함수를 통과시키면 다른 결과가 나와

**주요 활성화 함수 3가지:**

| 함수 | 특징 | 쓰이는 곳 |
|------|------|----------|
| **Sigmoid** | 출력을 0~1 사이로 변환. "확률"처럼 표현 | 이진 분류 출력층 |
| **ReLU** | 음수는 0, 양수는 그대로. 간단하고 강력! | 은닉층 (가장 많이 사용) |
| **Softmax** | 여러 값을 확률 분포로 변환 (합계=1) | 다중 분류 출력층 |

> 💡 **Sigmoid의 문제점**: 기울기 소실(Vanishing Gradient)
> - 값이 너무 크거나 작으면 기울기가 거의 0이 되어서 학습이 멈춰버려
> - 그래서 등장한 게 **ReLU**! 양수면 기울기가 항상 1이라서 학습이 잘 돼

### 1-5. 순전파 & 역전파: 학습의 핵심 메커니즘

> 자, 이제 딥러닝이 어떻게 학습하는지 전체 흐름을 볼게.

**순전파 (Forward Propagation):**
```
입력 → 계층1 → 계층2 → ... → 출력(예측값)
```
> 데이터를 앞으로 밀어서 예측값을 만드는 과정

**역전파 (Backpropagation):**
```
오차 계산 → 출력층 → 계층2 → 계층1 → 입력 방향으로 기울기 전파 → 가중치 수정
```
> 오차를 거꾸로 돌려보내서 각 가중치를 얼마나 바꿔야 하는지 계산하는 과정

> 비유하자면, **축구 경기 영상 리뷰**야 ⚽
> - 순전파: 경기 진행 (공격 → 슛 → 골 or 실패)
> - 역전파: 골키퍼가 골을 먹혔을 때 "내가 왼쪽으로 10cm 더 움직여야 했어!" 라고 복기하는 것
> - 이 복기를 통해 다음에는 더 잘 막을 수 있게 되는 거지!

### 1-6. PyTorch로 딥러닝 모델 만들기

> 이제 실제 코드로 어떻게 구현하는지 볼게!

**모델 정의 3단계:**
```python
# 1단계: nn.Module 상속
class MyModel(nn.Module):
    
    # 2단계: __init__에서 계층 정의
    def __init__(self):
        super().__init__()
        self.linear1 = nn.Linear(3, 2)  # 입력 3개 → 출력 2개
        self.relu = nn.ReLU()
        self.linear2 = nn.Linear(2, 1)  # 입력 2개 → 출력 1개
    
    # 3단계: forward에서 데이터 흐름 정의
    def forward(self, x):
        x = self.linear1(x)
        x = self.relu(x)
        x = self.linear2(x)
        return x
```

> ⚠️ **주의할 점**: 이전 계층의 `out_features`와 다음 계층의 `in_features`는 반드시 일치해야 해!
> → 소켓이랑 플러그 맞추는 것처럼, 크기가 안 맞으면 안 들어가 ㅇㅇ

**학습 루프 5단계:**
```python
for epoch in range(epochs):
    # 1. 순전파: 예측값 만들기
    pred = model(inputs)
    
    # 2. 손실 계산: 얼마나 틀렸나?
    loss = loss_fn(pred, targets)
    
    # 3. 기울기 초기화: 이전 기울기 지우기
    optimizer.zero_grad()
    
    # 4. 역전파: 기울기 계산
    loss.backward()
    
    # 5. 가중치 업데이트: 실제로 수정!
    optimizer.step()
```

> 💡 **optimizer.step()**이 진짜로 가중치를 바꾸는 순간!
> 이때 **학습률(Learning Rate)**이 적용되어 "얼마나 크게 수정할지" 결정해

## 📖 Part 2. CNN — 이미지 인식의 핵심

### 2-1. 왜 CNN이 필요할까? Linear Layer의 한계

> 자, 여기서 질문! 🤔
> **"왜 딥러닝 모델이 이미지를 인식하기 어려울까?"**
> 
> 기존 Linear Layer로 이미지를 처리하면 어떤 문제가 생길까?

**문제 1: 위치 의존성**
> 고양이가 사진 왼쪽에 있을 때랑 오른쪽에 있을 때, Linear Layer는 **완전히 다른 입력**으로 인식해
> → 같은 고양인데도 다른 걸로 판단해버리는 거야!

**문제 2: 파라미터 폭발 (차원의 저주)**
> 500×500 픽셀 컬러 이미지를 1차원으로 펼치면?
> → 250,000개의 입력 특성!
> → 은닉층 1,000개와 연결하면? **2억 5천만 개의 가중치** 필요 😱
> → 이건 계산량이 미친 듯이 많아져서 현실적으로 불가능해

> 그래서 등장한 게 **CNN (Convolutional Neural Network)**이야! 🎉

### 2-2. CNN의 핵심 아이디어: 특징 추출

> CNN의 기본 아이디어는 **인간의 시각 시스템을 모방**한 거야.
> 
> 우리가 사물을 볼 때, 처음부터 디테일을 보는 게 아니라:
> 1. 먼저 **큰 윤곽**을 파악하고 (Low-Level: 선, 모서리)
> 2. 그 다음 **중간 형태**를 인지하고 (Mid-Level: 곡선, 패턴)
> 3. 마지막으로 **구체적인 부분**을 파악해 (High-Level: 눈, 코, 바퀴)

> CNN도 똑같아! 여러 계층을 거치면서 점점 더 추상적이고 구체적인 특징을 학습해

**CNN의 구조:**
```
입력 이미지 → [Convolution + Pooling 반복] → Feature Maps → Fully Connected → 분류 결과
              ↑_________________________↑
                  Feature Extractor (특징 추출기)
```

> 💡 **과거 vs 현재:**
> - 과거: 사람이 직접 "이게 고양이 특징이야!" 하고 코딩
> - CNN: AI가 **스스로** 최적의 특징을 학습! → 이게 CNN의 가장 큰 장점!

### 2-3. 필터(Filter): 특징을 추출하는 렌즈

> CNN에서 가장 중요한 게 뭘까? 바로 **필터(Filter)**야!

> 비유하자면, **카메라 필터**랑 같아 📸
> - 인스타 필터 아니 ㅋㅋ 진짜 카메라 필터
> - 같은 장면도 UV 필터, 편광 필터, 컬러 필터를 끼우면 다르게 보이잖아
> - CNN의 필터도 마찬가지! 다양한 필터를 적용하면 이미지의 다른 특징이 추출돼

**컨볼루션 연산 과정:**
```
1. 필터(예: 3×3)를 이미지 위에 올려놓기
2. 필터와 겹치는 픽셀들을 각각 곱하기 (Element-wise Multiplication)
3. 곱한 값들을 모두 더하기 → 하나의 값!
4. 필터를 한 칸씩 이동(Sliding)하며 반복
5. 결과: 새로운 2D 배열 = Feature Map (특징 맵)
```

**예시:**
```
원본 이미지 영역:     필터:            계산:
[1, 2, 2]          [1,  0, -1]     1×1 + 2×0 + 2×(-1) +
[1, 5, 2]    ×     [1,  0, -1]  =  1×1 + 5×0 + 2×(-1) +  = -2
[3, 2, 3]          [1,  0, -1]     3×1 + 2×0 + 3×(-1)
```

> 💡 **핵심**: 필터의 값은 AI가 **학습을 통해 스스로** 만들어!
> 사람이 직접 설계하는 게 아니야. AI가 "이 필터가 고양이 귀를 잘 잡는다!" 하고 스스로 최적화하는 거야

### 2-4. 패딩(Padding): 크기 유지 & 모서리 보호

> 컨볼루션 연산을 하면 이미지 크기가 줄어들어 😢
> 6×6 이미지에 3×3 필터를 적용하면 → 4×4로 줄어!
> 
> 이걸 방지하기 위해 **패딩(Padding)**을 사용해.

> 비유하자면, **액자 테두리**야 🖼️
> - 사진 주변에 여백을 넣어서 원래 크기를 유지하는 거야
> - 주로 0으로 채우는 "Zero Padding"을 사용해

**패딩의 두 가지 목적:**
1. **Feature Map 크기 유지**: 컨볼루션으로 인한 크기 축소 방지
2. **모서리 특징 보호**: 패딩이 없으면 모서리 픽셀은 필터에 거의 안 닿아서 정보가 손실돼

> PyTorch에서는 `padding='same'`을 사용하면 자동으로 크기 맞춰줘 ㅇㅇ

### 2-5. 스트라이드(Stride): 필터의 보폭

> **스트라이드(Stride)**는 필터가 이동하는 간격이야.

- **Stride 1**: 한 칸씩 이동 → 디테일하게 특징 추출, 출력 크기 큼
- **Stride 2**: 두 칸씩 이동 → 빠르게 훑음, 출력 크기 작아짐

> 비유하자면, **책 읽기**야 📖
> - Stride 1: 한 줄씩 꼼꼼히 읽기
> - Stride 2: 두 줄씩 건너뛰며 빨리 읽기 (요약본 느낌)

> ⚠️ **주의**: Stride가 클수록 출력 Feature Map의 크기가 **더 작아져**!
> (강의에서 처음에 반대로 설명했다가 정정한 부분)

### 2-6. 커널 사이즈(Kernel Size): 필터의 크기

> **커널 사이즈**는 필터의 가로×세로 크기야.

| 커널 크기 | 장점 | 단점 |
|-----------|------|------|
| 큰 커널 (7×7) | 한 번에 넓은 영역 정보 수집 | 연산량 많음, 출력 크기 급감 |
| 작은 커널 (3×3) | 연산 효율적, 출력 크기 유지 | 한 번에 보는 영역 좁음 |

> 현대 CNN에서는 **3×3 커널을 여러 층 쌓는 방식**이 주류야!
> → 큰 커널 하나보다 작은 커널 여러 개가 더 효율적이거든

### 2-7. 풀링(Pooling): 다운샘플링

> **풀링(Pooling)**은 Feature Map의 크기를 줄이는 연산이야.

**Max Pooling (가장 많이 사용):**
```
2×2 영역에서 최댓값만 추출!

[1, 3]    → [3]
[2, 4]    

→ 가로/세로가 절반으로 줄어듦!
```

> 비유하자면, **사진 압축**이야 📷
> - 중요한 부분(가장 밝은 픽셀)은 유지하고, 나머지는 버리는 거야
> - 계산량도 줄고, 위치가 조금 바뀌어도 강건해져 (translational invariance)

## 📖 Part 3. PyTorch CNN 코드 — 14시, 15시 수업 내용

### 3-1. CNN 모델 구현하기

> 이제 실제로 CNN 모델을 코드로 만들어볼게!

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        
        # 첫 번째 합성곱: 3채널(RGB) → 32채널
        self.conv1 = nn.Conv2d(in_channels=3, out_channels=32, kernel_size=3)
        
        # 두 번째 합성곱: 32채널 → 64채널
        self.conv2 = nn.Conv2d(in_channels=32, out_channels=64, kernel_size=3)
        
        # Max Pooling: 2×2, 크기 절반으로
        self.pool = nn.MaxPool2d(kernel_size=2)
        
        # Flatten: 3D → 1D
        self.flatten = nn.Flatten()
        
        # Fully Connected: 분류기
        self.fc = nn.Linear(64 * 12 * 12, num_classes)
    
    def forward(self, x):
        x = F.relu(self.conv1(x))  # Conv → ReLU
        x = F.relu(self.conv2(x))  # Conv → ReLU
        x = self.pool(x)           # Pooling
        x = self.flatten(x)        # Flatten
        x = self.fc(x)             # 분류
        return x
```

**Feature Map 크기 변화 추적 (28×28 입력 기준):**
```
입력:        [3, 28, 28]
Conv1 (k=3): [32, 26, 26]    ← (28-3)/1+1 = 26
Conv2 (k=3): [64, 24, 24]    ← (26-3)/1+1 = 24
Pool (k=2):  [64, 12, 12]    ← 24/2 = 12
Flatten:     [9216]           ← 64 × 12 × 12
Linear:      [10]             ← 10개 클래스로 분류
```

### 3-2. Global Average Pooling (GAP) — Flatten의 대안

> 근데 위 방식에는 문제가 있어!
> Flatten을 쓰면 `64 × 12 × 12 = 9216`개의 값이 Linear 레이어로 들어가.
> → 파라미터가 너무 많아! 😱
> → 그리고 입력 이미지 크기가 바뀌면 이 숫자가 달라져서 모델을 다시 설계해야 해

> 이걸 해결하는 게 **Global Average Pooling (GAP)**이야!

**Flatten vs GAP 비교:**

| | Flatten | GAP |
|---|---------|-----|
| 방식 | 모든 값을 1D로 펼침 | 각 채널의 평균값 1개만 추출 |
| 파라미터 | 많음 (9216개) | 적음 (64개) |
| 입력 크기 | 고정 필요 | 유연함! |
| 오버피팅 | 발생 가능 | 방지 효과 |

**GAP 구현:**
```python
class SimpleCNN_GAP(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.conv1 = nn.Conv2d(3, 32, kernel_size=3)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3)
        self.pool = nn.MaxPool2d(kernel_size=2)
        
        # ✨ 핵심: Flatten 대신 GAP 사용!
        self.adapt_pool = nn.AdaptiveAvgPool2d(output_size=(1, 1))
        
        # in_features가 채널 수(64)로 고정!
        self.classifier = nn.Linear(in_features=64, out_features=num_classes)
    
    def forward(self, x):
        x = F.relu(self.conv1(x))
        x = self.pool(x)
        x = F.relu(self.conv2(x))
        x = self.pool(x)
        
        x = self.adapt_pool(x)           # [64, 12, 12] → [64, 1, 1]
        x = torch.flatten(x, start_dim=1) # [64, 1, 1] → [64]
        x = self.classifier(x)            # [64] → [10]
        return x
```

> 💡 **GAP의 가장 큰 장점**: 입력 이미지 크기가 32×32든 64×64든, 
> `AdaptiveAvgPool2d`가 항상 1×1로 압축해주니까 Linear 레이어의 in_features는 항상 64로 고정!
> → **입력 크기에 유연한 모델**이 되는 거야! 🎉

## 🎯 핵심 요약 (딱 5줄)

1. **딥러닝** = 인간의 뇌를 모방한 컴퓨터 학습 기술. 가중치를 업데이트하며 오차를 줄여나가는 과정이 학습이야.

2. **퍼셉트론**은 딥러닝의 최소 단위. 입력×가중치+편향을 활성화 함수에 통과시켜 출력을 만들어.

3. **순전파**로 예측하고, **역전파**로 오차를 줄이며, **경사하강법**으로 최적의 가중치를 찾아.

4. **CNN**은 이미지 인식의 핵심 기술. 필터(렌즈)로 특징을 추출하고, 패딩·스트라이드·풀링으로 효율적으로 처리해.

5. **GAP(Global Average Pooling)**는 Flatten의 대안으로, 파라미터를 줄이고 입력 크기에 유연한 모델을 만들어줘.
