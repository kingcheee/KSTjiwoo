*   **00:27 - 00:57**: 복잡한 수학적 설명보다는 핵심 개념의 의미와 중요성에 초점을 맞춰 학습에 도움이 되도록 정리하겠다고 언급합니다.
*   **01:14 - 01:35** (🖼️ **스크린샷 1-3: 퍼셉트론 개념**):
    *   **퍼셉트론(Perceptron)**의 개념을 소개합니다. 퍼셉트론은 여러 **입력 Feature (X1, X2... Xn)**에 **가중치 (W1, W2... Wn)**를 곱하고 합산(Weighted Sum)한 후 **Bias (편향)**를 더하고, 이를 **활성화 함수 (Activation Function, F)**를 통과시켜 최종 **출력 (Output)**을 만들어냅니다.
    *   핵심은 이 출력, 즉 **예측값(Prediction)**이 **정답(Correct Answer)**에 얼마나 근접하는지를 목표로 한다는 점입니다.
*   **01:35 - 02:07**: 심층 신경망(Deep Neural Network)의 기본 단위가 퍼셉트론이며, 궁극적으로 딥러닝이 하고자 하는 학습의 본질이 이 퍼셉트론의 원리와 같다고 설명합니다.
*   **02:08 - 02:38**: 퍼셉트론은 특정 예측값을 얻기 위해 필요한 입력 피처들을 가중치와 함께 합산하여 최종 출력값을 도출하는 과정이며, 이 가중치를 조절하는 것이 중요하다고 강조합니다.
*   **02:38 - 02:59**: **가중치(Weight)**를 잘 구성하고 조절하는 것이 곧 **'잘 학습한다'**는 의미이며, 이는 예측값과 정답 사이의 오차를 줄여나가는 과정과 직결됩니다.
*   **03:00 - 03:26**: 퍼셉트론의 핵심은 가중치(Weight)를 맞춰나가는 것이며, 이를 통해 예측값이 최종 정답에 가까워지도록 만드는 것이 목표입니다.
*   **03:26 - 03:48** (🖼️ **스크린샷 4: 퍼셉트론 학습**):
    *   최초의 가중치 값은 틀릴 수밖에 없으므로, **예측값**과 **실제값**의 **차이(오차)**를 계산하고, 이 오차를 줄일 수 있도록 **가중치(Weight)**를 계속 **변경(Update)**하는 과정이 반복됩니다.
*   **03:49 - 04:19** (🖼️ **스크린샷 5-6: 단순 선형 회귀 & 오차**):
    *   **단순 선형 회귀(Simple Regression)** 예시를 통해 예측값과 실제값 사이의 **오차(Error)** 개념을 시각적으로 설명합니다. 그래프의 점들은 실제 주택 가격(실제값)이고, 선은 예측 모델(예측값)입니다.
    *   예측 모델이 실제값과 얼마나 차이가 나는지(오차가 발생하는지)를 파악하는 것이 중요합니다.
*   **04:19 - 04:58**: 오차를 알아야만 가중치(W값)를 어떻게 변경해야 하는지 알 수 있습니다. 이 가중치를 계속 수정함으로써 예측값을 정답에 가깝게 만드는 것이 핵심입니다.
*   **04:59 - 05:39** (🖼️ **스크린샷 7: RSS와 MSE 공식**):
    *   **오차를 줄이는 방법**: 먼저 오차를 계산합니다.
    *   **RSS (Residual Sum of Squares)**: 예측값과 실제값 차이의 제곱을 모두 더한 값. `RSS = Σ(yi - ŷi)^2`
    *   **MSE (Mean Squared Error)**: RSS를 데이터 개수(N)로 나눈 값. 평균 제곱 오차라고도 하며, 딥러닝에서 가장 많이 사용되는 오차 지표 중 하나입니다. `MSE = (1/N) * Σ(yi - ŷi)^2`
*   **05:40 - 06:17** (🖼️ **스크린샷 8: 경사 하강법 개념**):
    *   **경사 하강법(Gradient Descent)**: 오차를 최소화하기 위한 방법입니다. 오차(비용 함수) 그래프에서 미분(기울기)을 통해 오차 값이 가장 작아지는 방향(최저점)으로 조금씩 이동하며 W(가중치) 값을 업데이트합니다.
*   **06:17 - 06:47** (🖼️ **스크린샷 9: 경사 하강법 요약**): 경사 하강법은 오차를 가장 줄일 수 있는 위치로 이동하는 것이며, W 파라미터가 많은 경우에도 간단하고 직관적인 비용 함수 최소화 솔루션을 제공합니다.
*   **06:48 - 07:32** (🖼️ **스크린샷 10-11: 미분을 통한 비용 함수 최소값 찾기**):
    *   미분은 증가 또는 감소하는 방향을 알려주며, 우리는 오차를 줄이기 위해 이 방향의 반대로 이동합니다.
    *   복잡한 미분 계산 과정을 모두 알 필요는 없으며, 미분이 최적의 지점(오차가 최소가 되는 지점)을 찾아주는 역할을 한다는 개념만 이해하면 충분하다고 강조합니다.
*   **07:43 - 08:43** (🖼️ **스크린샷 12-14: 심층 신경망**):
    *   **퍼셉트론**이 단일 계층인 반면, **심층 신경망(Deep Neural Network)**은 여러 계층(Hidden Layers)으로 구성되어 있습니다.
    *   각 계층의 뉴런들은 다음 계층의 모든 뉴런과 연결(Full Connection)되어 있어 훨씬 복잡한 학습이 가능합니다. 이 연결 하나하나마다 가중치(Weight)가 존재합니다.
    *   **가중치(Weight)**가 신경망의 복잡한 연결을 통해 계속해서 계산된다는 것은 곧 **'학습(Learning)'**이 이루어지고 있다는 의미입니다.
*   **08:44 - 11:53**: 입력되는 Feature 값들이 여러 가중치를 통해 다음 계층의 뉴런들로 전달되고, 이 과정이 반복되면서 신경망은 더욱 복잡한 특징을 학습합니다. 이처럼 여러 계층을 거치며 수많은 가중치들이 업데이트되는 것이 바로 **'심층 학습(Deep Learning)'**의 본질입니다.
    *   강사는 **가중치(Weight)**는 **학습(Learning)** 그 자체를 의미한다고 강조합니다.
*   **11:53 - 13:34**: 심층 신경망은 깊은 계층을 통해 복잡한 문제를 해결하며, 알렉스넷(AlexNet)이나 레스넷(ResNet)과 같은 현대의 CNN 모델들은 수십~수백 개의 계층을 가지고 있어 더 많은 가중치를 학습합니다.
*   **13:34 - 14:23**: 복잡한 계층 구조를 통해 더욱 정교한 가중치 학습이 이루어지며, 이를 통해 모델은 더욱 난이도 높은 문제를 해결할 수 있게 됩니다. 다시 한번, **가중치(Weight) = 학습(Learning)**임을 강조하며, 수학적 계산보다는 이 개념적 이해가 훨씬 중요하다고 역설합니다.
*   **14:46 - 15:34**: 계산의 '결과'보다는 '왜' 계산하는지, 이 계산을 통해 '무엇'을 얻고자 하는지, 즉 **목표(Purpose)**를 이해하는 것이 중요하다고 반복 강조합니다.
*   **15:35 - 16:01** (🖼️ **스크린샷 15-16: 심층 신경망 다이어그램 & Wx+b**): 퍼셉트론에 **Bias(b)**가 추가되어 `Wx+b` 형태로 계산됨을 다시 한번 언급합니다. (다이어그램에서는 생략될 수 있음)
*   **16:02 - 16:34** (🖼️ **스크린샷 17-18: 활성화 함수 개념**):
    *   **활성화 함수(Activation Function)**: 각 뉴런의 출력이 단순히 전달되는 것이 아니라 활성화 함수를 통과합니다.
    *   이 활성화 함수가 하는 역할은 네트워크에 **비선형성(Non-linearity)**을 부여하는 것입니다.
*   **16:35 - 17:28**: 활성화 함수는 단순한 '합격/불합격' 같은 이진적인 선형 결과를 넘어, **확률(Probability)**과 같은 세부적인 여지(leeway)를 제공하여 더욱 **디테일(Detailed)**한 학습이 가능하도록 합니다.
*   **17:28 - 18:22**: 즉, 활성화 함수는 **선형 회귀선(Linear Regression Line)** 같은 직선적인 결과가 아니라, 더욱 **비선형적인 곡선**을 만들어냄으로써 모델의 학습 능력을 향상시키는 역할을 합니다.
*   **18:22 - 19:35** (🖼️ **스크린샷 19-20: 역전파 개념**):
    *   **순전파(Forward Propagation)**: 입력 데이터가 신경망의 여러 계층을 거쳐 최종 예측값을 도출하는 과정. 이 과정에서 오차가 발생합니다.
    *   **역전파(Backpropagation)**: 순전파를 통해 발생한 오차를 기반으로, 각 가중치를 얼마나 수정해야 할지 결정하는 과정입니다. 최종 목표는 가중치를 수정하여 오차를 최소화하는 것입니다.
*   **19:35 - 20:44**: 순전파가 오차의 크기를 알려준다면, 역전파는 그 오차를 줄이기 위해 가중치(Weight)를 어느 방향으로, 얼마나 수정해야 하는지를 계산합니다. 역전파 과정은 미분 연쇄 법칙(Chain Rule)이 적용되는 복잡한 계산을 포함하지만, 개념적 이해가 중요합니다.
*   **20:44 - 21:14** (🖼️ **스크린샷 21-22: 활성화 함수 종류**):
    *   **Sigmoid(시그모이드)**: S자 형태의 곡선으로, 출력값을 0~1 사이로 변환합니다. 이진 분류(Binary Classification)에 사용되지만, **기울기 소실(Vanishing Gradient)** 문제 발생 가능성이 있습니다.
    *   **ReLU(렐루)**: 은닉층에서 가장 많이 사용되는 활성화 함수로, 기울기 소실 문제를 해결하며 계산 효율성이 높습니다.
*   **21:14 - 22:23** (🖼️ **스크린샷 23: ReLU 함수 미분**): Sigmoid의 기울기 소실 문제에 대한 해결책으로 ReLU 함수가 등장했으며, 현재 가장 많이 사용되는 함수입니다.
*   **22:24 - 23:08** (🖼️ **스크린샷 24-25: Softmax 함수 개념 & 계산**):
    *   **Softmax(소프트맥스)**: 다중 분류(Multi-class Classification) 문제에서 사용되며, 여러 개의 출력값을 확률 분포(총합이 1이 되도록)로 변환합니다.
    *   이를 통해 각 클래스에 속할 확률을 직관적으로 파악하고 판단하기 용이해집니다.
*   **23:09 - 24:22**: 시그모이드는 이진 분류, 소프트맥스는 다중 분류에 주로 사용됩니다. 이 둘은 최종 출력 계층에서 사용됩니다.
*   **24:22 - 25:44**: 소프트맥스는 모델의 최종 아웃풋을 0과 1사이의 확률 분포로 변환하여, 여러 카테고리 중 어떤 것이 정답일 확률이 높은지 한눈에 쉽게 판단할 수 있도록 돕습니다.
*   **25:45 - 26:45** (🖼️ **스크린샷 26-28: PyTorch 신경망 모델**):
    *   **PyTorch**는 딥러닝 모델 구축을 위한 다양한 라이브러리(torch, autograd, nn, optim, utils, torchvision)를 제공합니다.
    *   **`nn.Module`**: 신경망을 구성하는 기본 단위로, 모든 사용자 정의 모델은 이 클래스를 상속받습니다.
    *   **`nn.Linear`**: 완전 연결(Fully Connected) 계층을 생성합니다.
    *   **`nn.ReLU`, `nn.Sigmoid`**: 활성화 함수 계층을 제공합니다.
    *   **`optim`**: SGD, Adam 등 최적화(Optimizer) 알고리즘을 제공합니다.
*   **26:46 - 27:41** (🖼️ **스크린샷 29-31: nn.Module 클래스 정의**):
    *   **PyTorch 모델 생성 방법:**
        1.  **`nn.Module` 상속**: `class MyModel(nn.Module):`
        2.  **`__init__` 함수 정의**: `def __init__(self, ...):`
            *   **부모 생성자 호출**: `super().__init__()`
            *   **계층(Layer) 정의**: `self.linear1 = nn.Linear(in_features, out_features)`
        3.  **`forward` 함수 정의**: `def forward(self, x):` (순전파 로직 구현)
*   **27:41 - 29:00** (🖼️ **스크린샷 32-35: 계층 연결 시 Feature 개수 일치**):
    *   **`nn.Linear`** 계층을 정의할 때, 이전 계층의 `out_features`와 현재 계층의 `in_features`는 반드시 일치해야 합니다. (예: `nn.Linear(3, 2)` 다음에 `nn.Linear(2, 1)`이 와야 함).
    *   `forward` 함수에서는 정의된 계층들을 순서대로 연결하여 데이터 흐름을 만듭니다.
*   **29:01 - 29:55** (🖼️ **스크린샷 36: `forward` 함수 로직**): `forward` 함수는 입력 `x`를 받아 `self.linear_01(x)`, `self.relu_01(x)`, `self.linear_02(x)` 순으로 처리하여 최종 `output`을 반환합니다. 이것이 **순방향 전파**입니다.
*   **29:55 - 30:08**: 모델 생성의 핵심 단계를 다시 한번 요약: `nn.Module` 상속, `__init__` 및 `forward` 함수 정의, `super().__init__()` 호출, 필요한 계층 정의, 그리고 `in_features`와 `out_features` 개수 맞추기.
*   **30:09 - 30:58** (🖼️ **스크린샷 37-38: PyTorch 계층 종류 & `nn.Conv2d`, `nn.MaxPool2d`**): `nn.Linear`, `nn.ReLU`, `nn.Sigmoid` 외에 `nn.Conv2d` (합성곱 계층)와 `nn.MaxPool2d` (풀링 계층) 등 다양한 계층이 있으며, 이들은 주로 이미지 처리(CNN)에 사용됩니다.
*   **30:58 - 33:14** (🖼️ **스크린샷 39-41: 계층 연결 시 Feature 개수 일치 예시**):
    *   `nn.Linear(in_features=3, out_features=3)` -> `nn.Linear(in_features=3, out_features=2)` -> `nn.Linear(in_features=2, out_features=1)`

### 3. 핵심 개념 및 코드/공식 정리

**1. 퍼셉트론 (Perceptron)**
*   **개념**: 딥러닝의 가장 기본적인 구성 요소. 여러 입력에 가중치를 곱하고 합산 후 편향을 더해 활성화 함수를 통과시켜 출력을 생성.
*   **학습 목표**: 예측값을 실제값에 최대한 가깝게 만드는 것.
*   **핵심**: 가중치(Weight)를 조절하여 예측 오차를 줄여나가는 과정.

**2. 오차 / 손실 함수 (Loss Function)**
*   **역할**: 모델의 예측값과 실제값 사이의 차이를 정량화. 이 값을 최소화하는 것이 학습의 목표.
*   **MSE (Mean Squared Error)**:
    *   `MSE = (1/N) * Σ(yi - ŷi)^2`
    *   회귀(Regression) 문제에서 주로 사용되는 손실 함수. 예측값과 실제값 차이의 제곱 평균.

**3. 경사 하강법 (Gradient Descent)**
*   **개념**: 손실 함수(Cost Function)의 최저점을 찾아 가중치를 업데이트하는 최적화 알고리즘.
*   **원리**: 손실 함수의 기울기(Gradient)를 계산하여, 기울기의 반대 방향(오차가 감소하는 방향)으로 가중치 값을 조금씩 이동(업데이트).
*   **목표**: 오차를 최소화하는 최적의 가중치 조합을 찾는 것.

**4. 활성화 함수 (Activation Function)**
*   **역할**: 신경망에 비선형성을 부여하여 복잡한 패턴 학습 가능하게 함.
*   **Sigmoid (시그모이드)**:
    *   `f(x) = 1 / (1 + e^-x)`
    *   출력 범위: (0, 1). 이진 분류(Binary Classification)의 출력 계층에 사용.
    *   문제점: 기울기 소실(Vanishing Gradient) 문제 발생 가능성.
*   **ReLU (Rectified Linear Unit)**:
    *   `f(x) = max(0, x)`
    *   출력 범위: [0, ∞). 은닉층(Hidden Layer)에서 가장 많이 사용. 기울기 소실 문제 완화.
*   **Softmax (소프트맥스)**:
    *   `P(class_i) = e^(zi) / Σ(e^zj)`
    *   출력 범위: 각 클래스 확률이 (0, 1) 사이이며, 모든 클래스 확률의 합은 1.
    *   다중 분류(Multi-class Classification)의 출력 계층에 사용.

**5. 순전파 (Forward Propagation) & 역전파 (Backpropagation)**
*   **순전파**: 입력 데이터를 신경망에 통과시켜 최종 예측값을 얻는 과정.
*   **역전파**: 순전파로 얻은 예측 오차를 기반으로, 각 가중치에 대한 손실 함수의 기울기(Gradient)를 계산하여 가중치를 업데이트하는 과정. 학습의 핵심.

**6. PyTorch 모델 구축 (`nn.Module` 클래스)**

*   **모델 정의 코드 구조**:
    ```python
    import torch.nn as nn

    class CustomModel(nn.Module):
        def __init__(self, in_dim, num_classes):
            super().__init__() # 부모 클래스 생성자 호출
            # 계층(Layer) 정의
            self.linear1 = nn.Linear(in_dim, 64) # 입력층 -> 은닉층 1
            self.relu1 = nn.ReLU()             # 활성화 함수
            self.linear2 = nn.Linear(64, 32)   # 은닉층 1 -> 은닉층 2
            self.relu2 = nn.ReLU()             # 활성화 함수
            self.linear3 = nn.Linear(32, num_classes) # 은닉층 2 -> 출력층

        def forward(self, x):
            # 순전파(Forward Propagation) 로직 구현
            x = self.linear1(x)
            x = self.relu1(x)
            x = self.linear2(x)
            x = self.relu2(x)
            output = self.linear3(x)
            return output
    ```
*   **`nn.Linear(in_features, out_features)`**: 완전 연결 계층. `in_features`는 이전 계층의 `out_features`와 일치해야 함.
*   **`nn.Module` 상속**: 사용자 정의 모델은 반드시 `nn.Module`을 상속받아야 함.
*   **`super().__init__()`**: 부모 클래스인 `nn.Module`의 생성자를 호출해야 함.

**7. PyTorch 학습 루프 (Training Loop)**

*   **학습 루프 코드**:
    ```python
    # 모델, 손실함수, 옵티마이저 정의 예시
    model = CustomModel(X_train.shape[1], 1) # X_train.shape[1]은 입력 특성 수
    criterion = nn.MSELoss() # 손실 함수 (MSE for regression)
    optimizer = torch.optim.Adam(model.parameters(), lr=0.01) # 옵티마이저 (Adam)

    epochs = 200 # 학습 횟수

    for epoch in range(epochs):
        # --- Train ---
        model.train() # 모델을 학습 모드로 설정
        optimizer.zero_grad() # 이전 epoch의 기울기 초기화
        pred = model(X_train_t) # 순전파 (Forward Pass)
        loss = criterion(pred, y_train_t) # 손실 계산
        loss.backward() # 역전파 (Backpropagation) - 기울기 계산
        optimizer.step() # 가중치 업데이트

        # --- Validation ---
        model.eval() # 모델을 평가 모드로 설정 (기울기 계산 불필요)
        with torch.no_grad(): # 기울기 계산 비활성화
            val_pred = model(X_test_t)
            val_loss = criterion(val_pred, y_test_t)
    ```
