### 3. 핵심 개념 및 코드/공식 정리

#### 1. PyTorch 신경망 모델 기본 구조
- **입력 (Input Feature)**: 이미지 픽셀 값 등을 벡터 형태로 변환하여 신경망에 주입 (예: 28x28 이미지 -> 784개 피처).
- **선형 계층 (Linear Layer)**: 입력 특성에 가중치를 곱하고 편향을 더하는 기본적인 신경망 연산.
- **활성화 함수 (Activation Function)**: 신경망의 비선형성을 추가하여 복잡한 패턴 학습 가능 (예: ReLU).
- **출력 계층 (Output Layer)**: 최종 예측값을 도출하는 계층.
- **소프트맥스 (Softmax)**: (언급만 됨) 다중 클래스 분류 문제에서 출력값을 확률 분포로 변환하는 함수.

#### 2. `torch.nn.Module`
- PyTorch에서 모든 신경망 모듈의 **기반이 되는 클래스**입니다.
- 사용자 정의 신경망 모델을 만들려면 반드시 `nn.Module`을 상속받아야 합니다.
- 상속을 통해 `parameters()`, `eval()`, `train()`, `to()` 등 모델 관리에 필요한 다양한 메서드를 자동으로 사용할 수 있게 됩니다.

#### 3. 객체 지향 프로그래밍(OOP)의 상속 (Inheritance)
- **정의**: 기존 클래스(부모/상위 클래스)의 속성과 메서드를 새로운 클래스(자식/하위 클래스)가 물려받아 사용하는 개념.
- **목적**:
    - **코드 재사용성**: 공통적인 코드를 반복해서 작성할 필요 없이, 부모 클래스에서 한 번 정의하고 자식 클래스에서 재사용.
    - **유지보수 용이성**: 공통 기능 변경 시 부모 클래스만 수정하면 모든 자식 클래스에 반영.
    - **확장성**: 기존 기능을 변경하지 않고 새로운 기능을 추가하여 모델을 확장.
- **`super().__init__()`**: 자식 클래스의 `__init__` 메서드 내에서 부모 클래스의 `__init__` 메서드를 호출하는 구문. 상속받은 부모 클래스의 초기화 작업을 반드시 수행해야 하므로 필수적입니다.

#### 4. 클래스와 인스턴스
- **클래스 (Class)**: 객체를 생성하기 위한 **설계도** 또는 **틀** (예: 붕어빵 틀).
- **인스턴스 (Instance)**: 클래스를 바탕으로 메모리에 실제로 생성된 **객체** (예: 붕어빵).
- **`__init__`**: 클래스의 **생성자** 함수. 객체(인스턴스)가 생성될 때 자동으로 호출되어 객체의 초기 상태를 설정하고 속성을 정의합니다.

#### 5. `self` 매개변수
- 파이썬 클래스의 메서드에서 항상 첫 번째 매개변수로 사용되며, 해당 메서드를 호출한 **객체(인스턴스) 자신**을 참조합니다.
- `self.변수명` 형태로 객체(인스턴스) 고유의 속성을 저장하고 접근하는 데 사용됩니다.

#### 6. `forward` 함수
- `nn.Module`을 상속받은 클래스에서 **순전파(forward propagation)** 로직을 정의하는 메서드입니다.
- 입력 데이터 `x`가 신경망의 각 계층을 어떻게 통과하여 최종 출력을 생성하는지 기술합니다.
- 직접 호출하는 것이 아니라, `model(input_data)`와 같이 모델 객체를 함수처럼 호출하면 자동으로 `forward` 메서드가 실행됩니다.
