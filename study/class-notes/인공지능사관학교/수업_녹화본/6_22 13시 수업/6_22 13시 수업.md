### 3. 핵심 개념 및 코드/공식 정리

**Convolutional Neural Networks (CNNs) 핵심 요소**

1.  **Feature Map (피처 맵)**
    *   **정의**: 컨볼루션 연산의 결과로 생성되는 출력 데이터. 입력 이미지의 특정 특징(선, 모서리, 질감 등)을 강조하여 표현합니다.
    *   **역할**: 이미지 분석 및 분류를 위한 핵심 정보로 활용됩니다.

2.  **Filter (필터) / Kernel (커널)**
    *   **정의**: 입력 이미지 위에 슬라이딩하며 컨볼루션 연산을 수행하는 작은 행렬입니다.
    *   **역할**: 이미지에서 특정 특징을 추출하는 렌즈와 같습니다. 필터의 값들은 모델 학습을 통해 최적화됩니다.
    *   **Filter vs. Kernel**: 넓은 의미에서 '필터'는 '커널'을 포함합니다. 특히 다채널 이미지 처리 시, 필터는 여러 개의 2차원 커널(채널)들로 구성될 수 있습니다.
    *   **Kernel Size (커널 크기)**: 커널의 가로/세로 크기(예: 3x3, 5x5).
        *   **장점**: 크면 클수록 더 넓은 영역을 한 번에 보므로 더 많은 Feature 정보를 추출할 수 있습니다.
        *   **단점**: 크면 클수록 연산량이 증가하고, 파라미터 수가 늘어나며, 출력 Feature Map의 크기가 감소합니다.

3.  **Convolution Operation (컨볼루션 연산)**
    *   **정의**: 필터가 입력 이미지 위를 일정한 간격으로 슬라이딩하면서, 필터와 겹치는 입력 영역의 픽셀 값들을 곱하고 합산하여 하나의 출력값을 생성하는 과정입니다.
    *   **출력 Feature Map 크기 계산 공식 (일반적인 경우, Stride 1, Padding 없음)**:
        `출력_크기 = (입력_크기 - 커널_크기) / Stride + 1`
    *   **예시 연산 (슬라이드 05:02 참고):**
        ```
        Input: [ [1, 2, 2, 0],
                 [1, 5, 2, 1],
                 [3, 2, 3, 4],
                 [6, 9, 2, 4] ]
        Filter: [ [1,  0, -1],
                  [1,  0, -1],
                  [1,  0, -1] ]
        
        첫 번째 출력값 계산 (슬라이드 05:02, 05:32):
        (1*1) + (2*0) + (2*-1) +
        (1*1) + (5*0) + (2*-1) +
        (3*1) + (2*0) + (3*-1) = 1 + 0 - 2 + 1 + 0 - 2 + 3 + 0 - 3 = -2
        ```
        (강사는 강의 중 계산식의 일부에 대한 구두 정정을 진행했습니다.)

4.  **Padding (패딩)**
    *   **정의**: 컨볼루션 연산을 수행하기 전에 입력 이미지의 가장자리에 추가 픽셀(주로 0)을 채워 넣는 기법입니다.
    *   **주요 목적 및 이점**:
        1.  **Feature Map 크기 유지**: 컨볼루션 연산 시 Feature Map이 계속 줄어드는 것을 방지하여, 출력 Feature Map의 크기를 입력 Feature Map의 크기와 동일하게 유지할 수 있습니다 (예: PyTorch의 `padding='same'`).
        2.  **모서리(가장자리) 특징 보존 및 강화**: 패딩을 통해 커널이 이미지의 가장자리 픽셀 위를 여러 번 지나갈 수 있게 되어, 모서리 주변의 특징 정보를 더 많이 추출하고 강화할 수 있습니다.
    *   **크기 유지 예시 (슬라이드 12:34, 13:05 참고):**
        *   5x5 입력, 3x3 커널, Stride 1 (패딩 없음) -> `(5-3)/1 + 1 = 3` (3x3 출력)
        *   5x5 입력을 1칸 패딩하여 7x7로 만들고, 3x3 커널, Stride 1 적용 -> `(7-3)/1 + 1 = 5` (5x5 출력) -> 원본 크기 유지

5.  **Stride (스트라이드)**
    *   **정의**: 필터가 입력 이미지 위를 슬라이딩할 때 이동하는 간격(픽셀 수)입니다.
    *   **Stride 1 (기본값)**:
        *   **특징**: 필터가 한 칸씩 이동합니다.
        *   **결과**: 출력 Feature Map의 크기가 상대적으로 크고, 원본 이미지의 정보를 더 세밀하게 보존합니다.
        *   **단점**: 연산량이 많아집니다.
    *   **Stride > 1 (예: Stride 2)**:
        *   **특징**: 필터가 두 칸 이상씩 이동합니다.
        *   **결과**: 출력 Feature Map의 크기가 작아지고, 정보의 밀도가 낮아집니다 (다운샘플링 효과).
        *   **장점**: 연산량이 크게 줄어들어 처리 속도가 향상됩니다. 주로 고수준의 추상적인 특징을 추출하거나 연산 효율이 중요할 때 사용됩니다.
