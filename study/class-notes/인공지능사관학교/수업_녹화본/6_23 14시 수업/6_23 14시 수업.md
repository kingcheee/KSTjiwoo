*   토큰화(Tokenization) 작업의 상세 과정:
    1.  텍스트 정규화(Text Normalization): 소문자 변환, 불필요한 공백/특수문자/이모지 제거.

    4.  Post 토큰화 작업: 특수 토큰 추가(예: `[CLS]`, `[SEP]`), OOV(Out-Of-Vocabulary) 처리, Padding 적용, Tensor 변환.

    *   `text = text.lower()`: 텍스트를 소문자로 변환합니다.
    *   `text = re.sub(r'[^a-z0-9\s]', '', text)`: 알파벳, 숫자, 공백을 제외한 모든 문자를 제거합니다.
    *   `tokens = text.split()`: 공백 기준으로 텍스트를 토큰 리스트로 분리합니다.
    *   `return tokens`를 통해 토큰 리스트를 반환합니다.
    *   `normalized_tokens = normalize(test)`를 실행하여 결과 `['hello', 'woird', 'i', 'am', 'learning', 'nlp']`를 확인합니다.
*   **BERT 토크나이저 및 모델 로드**:
    *   `from transformers import AutoTokenizer, AutoModel`
    *   `tokenizer = AutoTokenizer.from_pretrained('bert-base-uncased')`
    *   `model = AutoModel.from_pretrained('bert-base-uncased')`
*   **텍스트 임베딩 및 분류 실습 마무리**:
    *   `inputs = tokenizer(normalized_tokens, return_tensors='pt', is_split_into_words=True)`: `is_split_into_words=True` 옵션을 사용하여 이미 분리된 토큰 리스트를 입력으로 받습니다.
    *   `outputs = model(**inputs)`
    *   `sentence_embedding = outputs.pooler_output[0]`
    *   `output = classifier(sentence_embedding)`
    *   `probabilities = softmax(output)`
    *   `predicted_class = torch.argmax(probabilities)`
    *   `print(labels[predicted_class])`를 통해 예측된 클래스를 출력하며 텍스트 분류 실습을 마무리합니다.
    *   이전과 달리 `is_split_into_words=True`를 사용했으므로, `normalized_tokens`를 직접 `tokenizer`에 전달합니다.

## 3. 핵심 개념 및 코드/공식 정리

### 1. NLP 발전 단계

*   **2010년 이전**: 전통적인 방식 (규칙 기반, 통계 기반)
    *   언어 전문가의 고품질 샘플 데이터 확보 중요.
    *   언어학 기반 규칙 시스템 (`rule-based program`).
    *   말뭉치(Corpus) 기반 통계 모델 (`Statistical Machine Translation`).
*   **2010년 이후 (딥러닝 기반)**:
    *   **Word Embedding**: 전체 입력 문장 단위로 처리.
    *   **어순, 단어의 의미, 문맥 파악** 등을 스스로 학습.
    *   **언어 전문가 불필요**.
    *   **모델**: Bidirectional Recurrent Neural Network (RNN), Encoder-decoder (seq2seq), Attention model, Transformer model, BERT/GPT-3 등.

### 2. 딥러닝 기반 NLP의 주요 요소

*   **Tokenization(토큰화)**:
    *   문장을 작은 토큰(단어, 단어 조각)으로 분할하여 단어 사전(어휘 사전)을 구축하는 과정.
    *   **과정**: 텍스트 정규화 (소문자 변환, 불필요 문자 제거) → 토큰 분할 (공백, 문장 부호 기준) → 어휘 사전 매핑 (토큰을 Index 값으로 매핑) → 특수 토큰 추가, Padding, Tensor 변환.
    *   **예시**: `"Hello, world! I am learning NLP."` → `hello world i am learning nlp` → `['hello', 'world', 'i', 'am', 'learning', 'nlp']`
    *   **서브워드(Subword) 토큰화**: "unhappiness" → `['un', '##happi', '##ness']` (미등록 단어 문제 해결, 일반화).
*   **Embedding(임베딩)**:
    *   토큰(단어)을 다차원 벡터로 표현하는 과정.
    *   컴퓨터가 이해할 수 있는 숫자 형태로 단어의 의미를 반영하는 벡터를 생성.
    *   임베딩된 벡터는 텍스트 분류, 감성 분석 등 다양한 NLP 작업의 입력으로 사용.
*   **딥러닝 모델**:
    *   임베딩 데이터를 입력받아 문장 내 문맥을 이해하고 학습할 수 있는 모델.
    *   예시: RNN, LSTM, Transformer (Attention 메커니즘 사용), BERT/GPT (트랜스포머 기반).

### 3. 주요 도구 및 개념

*   **Hugging Face**: 미리 학습된 NLP 모델(BERT, GPT 등)과 토크나이저를 제공하는 플랫폼. (`transformers` 라이브러리 사용).
*   **`AutoTokenizer.from_pretrained('model_name')`**: 모델에 맞는 토크나이저를 로드.
*   **`AutoModel.from_pretrained('model_name')`**: 미리 학습된 모델을 로드.
*   **`tokenizer(text, return_tensors='pt')`**: 텍스트를 토큰화하고 PyTorch 텐서로 변환. `input_ids`, `attention_mask` 등의 출력을 포함.
*   **`model(**inputs)`**: 토큰화된 입력을 모델에 전달하여 임베딩 및 기타 출력을 얻음.
*   **`outputs.last_hidden_state`**: 각 토큰에 대한 문맥을 반영한 임베딩 벡터. (예: `torch.Size([1, 8, 768])` → 1문장, 8토큰, 768차원).
*   **`outputs.pooler_output`**: `[CLS]` 토큰의 임베딩으로, 문장 전체의 의미를 요약하는 데 자주 사용됨. (예: `torch.Size([1, 768])`).
*   **기울기 소실(Gradient Vanishing)**: 딥러닝 모델 학습 시, 계층이 깊어질수록 기울기가 0에 수렴하여 가중치 업데이트가 거의 이루어지지 않는 현상. ReLU 등의 활성화 함수로 완화.
*   **객체 지향 프로그래밍(OOP)과 상속**:
    *   OOP는 코드를 모듈화하고 재사용성을 높이는 프로그래밍 패러다임.
    *   상속을 통해 기존 클래스의 속성과 메서드를 물려받아 새로운 클래스를 만들 수 있음.
    *   NLP에서 토큰이나 임베딩과 같은 기본 구성 요소를 상속받아 특정 기능을 추가하거나 확장하는 데 활용. 이는 '계좌' 예시(마이너스 통장, 입금, 체크 카드 기능 상속)를 통해 설명됨.
    *   오늘날 대부분의 프로그래밍 언어는 객체 지향을 지원하며, NLP 개발에 필수적.
