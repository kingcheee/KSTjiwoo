### 3. 핵심 개념 및 코드/공식 정리

*   **CNN 모델 아키텍처 (`SimpleCNN_02`)**
    *   `self.conv_1 = nn.Conv2d(in_channels=NUM_INPUT_CHANNELS, out_channels=32, kernel_size=3, stride=1, padding=1)`
        *   입력 채널: `NUM_INPUT_CHANNELS` (3), 출력 채널: 32, 커널 크기: 3x3, 스트라이드: 1, 패딩: 1. 패딩=1은 컨볼루션 후 이미지 크기 유지.
    *   `self.conv_2 = nn.Conv2d(in_channels=32, out_channels=64, kernel_size=3, stride=1, padding=1)`
        *   `conv_1`의 출력 채널인 32를 입력으로 받아 64개의 특징 맵 생성.
    *   `self.conv_block = nn.Sequential(...)`
        *   `nn.Sequential`을 사용하여 여러 컨볼루션 및 풀링 계층을 그룹화한 서브 모듈.
        *   `nn.Conv2d(in_channels=64, out_channels=128, kernel_size=3, padding='same')` (x2)
        *   `nn.MaxPool2d(kernel_size=2)`
    *   `self.pool = nn.MaxPool2d(kernel_size=2)`
        *   공간적 크기를 절반으로 줄임 (e.g., 32x32 -> 16x16).
    *   `self.adapt_pool = nn.AdaptiveAvgPool2d(output_size=(1,1))`
        *   어떤 크기의 입력이든 각 채널을 1x1 크기로 평균화하여 출력. 이는 다양한 입력 이미지 크기에 대응하는 유연성을 제공.
    *   `self.flatten = nn.Flatten()`
        *   다차원 텐서를 1차원 벡터로 변환하여 `nn.Linear` 계층의 입력으로 사용 (`start_dim=1`은 배치 차원 제외).
    *   `self.classifier = nn.Linear(in_features=128, out_features=NUM_CLASSES)`
        *   128개의 특징을 입력받아 `NUM_CLASSES` (10)개의 최종 분류 클래스로 출력.

*   **데이터셋 로딩 및 전처리**
    *   `torchvision.datasets.CIFAR10`: CIFAR-10 데이터셋 (10개 클래스, 32x32 컬러 이미지, 6만 개).
    *   `torchvision.transforms.ToTensor()`: 이미지 데이터를 PyTorch 텐서로 변환.
    *   `torch.utils.data.random_split(train_dataset, [tr_size, val_size])`: 학습 데이터를 학습용(85%)과 검증용(15%)으로 분할.
    *   `torch.utils.data.DataLoader(dataset, batch_size=32, shuffle=..., num_workers=4)`:
        *   `batch_size=32`: 32개 이미지씩 묶어 처리.
        *   `shuffle=True`: 학습 데이터는 에포크마다 섞음 (일반화 증진).
        *   `shuffle=False`: 검증/테스트 데이터는 순서 고정 (일관된 평가).
        *   `num_workers=4`: 데이터를 병렬로 로드하여 로딩 속도 향상.

*   **학습 과정 (`Trainer` 클래스)**
    *   `__init__(self, model, loss_fn, optimizer, train_loader, val_loader, device=None)`: 모델, 손실 함수, 옵티마이저, 데이터 로더, 디바이스를 초기화.
    *   `train_epoch(self, epoch)`:
        *   `self.model.train()`: 모델을 학습 모드로 설정.
        *   `optimizer.zero_grad()`: 이전 스텝에서 계산된 기울기 초기화.
        *   `outputs = self.model(inputs)`: 순전파(Forward Pass)로 예측값 계산.
        *   `loss = self.loss_fn(outputs, targets)`: 손실 함수로 손실 값 계산.
        *   `loss.backward()`: 역전파(Backpropagation)로 각 파라미터의 기울기 계산.
        *   `optimizer.step()`: 계산된 기울기를 바탕으로 모델 파라미터 업데이트.
    *   `validate_epoch(self, epoch)`:
        *   `self.model.eval()`: 모델을 평가 모드로 설정 (드롭아웃, 배치 정규화 등 비활성화).
        *   `with torch.no_grad()`: 기울기 계산을 비활성화하여 메모리 사용량 줄이고 속도 향상.
        *   역전파 및 옵티마이저 스텝은 수행하지 않음.
    *   `fit(self, epochs)`:
        *   지정된 `epochs` 수만큼 `train_epoch`와 `validate_epoch`를 반복 수행.
        *   각 에포크의 학습 및 검증 손실/정확도(`history` 딕셔너리)를 기록.

*   **그래프 시각화 (`show_history`)**
    *   `import matplotlib.pyplot as plt`: 시각화 라이브러리.
    *   학습 과정에서 기록된 `history` 데이터를 사용하여 학습/검증 손실 및 정확도 변화를 그래프로 표현, 과적합(overfitting) 여부 판단에 도움.
