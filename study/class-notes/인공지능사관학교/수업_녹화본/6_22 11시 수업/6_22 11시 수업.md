## 6_22 11시 수업 학습 노트: 컨볼루션 연산의 핵심 이해

### 1. 강의 핵심 요약

오늘의 6_22 11시 수업은 컨볼루션 연산(Convolutional Operation)의 기본적인 작동 방식과 그 핵심 구성 요소인 필터(커널), 스트라이드, 패딩의 중요성을 심층적으로 다루었습니다. 강사님은 특히 이 세 가지 요소가 이미지 처리 과정에서 데이터 손실을 최소화하고 특징을 효과적으로 추출하며, 최종적으로 이미지 분류 성능에 어떻게 영향을 미치는지를 강조했습니다. 궁극적으로 CNN에서 AI가 최적의 필터 값을 스스로 학습하는 방식과 이러한 파라미터 설정이 모델 성능에 미치는 영향을 이해하는 것이 수업의 주요 목표였습니다.

### 2. 화면 연계 타임라인 노트

- **00:00:00 - 00:00:30 (화면 캡처: Input, Filter, Output Diagram)**
    - 컨볼루션 연산(Convolutional Operation)의 기본 개념 소개.
    - 입력 이미지(예: 8x8)에 3x3 필터(커널)를 적용하면 출력이 극적으로 작아질 수 있음을 설명. (예: 3x3으로 줄어듦).
    - 화면의 그림은 8x8 입력에 3x3 필터를 적용하여 3x3 피처 맵이 생성되는 과정을 시각화.

- **00:00:30 - 00:01:00 (화면 캡처: Padding 적용된 Input Diagram)**
    - 이미지 크기 축소 문제를 해결하기 위한 '패딩(Padding)'의 필요성 설명.
    - 패딩은 입력 이미지 주변에 0값을 추가하여 원본 이미지의 크기가 출력에서 급격히 줄어드는 것을 방지.
    - 강사님은 패딩을 통해 원본 이미지의 크기가 유지될 수 있는지 질문.

- **00:01:00 - 00:01:30 (화면 캡처: Padding 적용 후 Output Diagram)**
    - 패딩이 적용된 입력 이미지에 필터를 슬라이딩하는 과정을 시연.
    - 패딩 덕분에 출력 피처 맵의 크기가 (패딩이 없을 때보다) 더 커지고, 원본 이미지의 정보가 더 많이 보존됨.
    - 강사님은 4x4 이미지에 대한 설명으로 패딩의 효과를 다시 한번 강조.

- **00:01:30 - 00:02:00 (화면 캡처: Filter Sliding Visualization)**
    - 필터 슬라이딩 시 중앙에 위치한 픽셀들은 여러 번 연산에 포함되지만, 모서리(가장자리) 픽셀들은 한두 번만 포함되는 문제 제기.
    - 이로 인해 모서리 부분의 정보가 충분히 반영되지 않을 수 있음을 지적.

- **00:02:00 - 00:02:30 (화면 캡처: Filter Sliding Visualization)**
    - 패딩이 모서리 픽셀의 정보 손실을 줄이는 역할을 함을 설명.
    - 패딩을 통해 모서리 픽셀도 필터 연산에 여러 번 참여하게 되어, 피처 맵에 더 잘 반영될 수 있음.

- **02:30 - 03:00 (화면 캡처: Filter Sliding Visualization - 최종 결과)**
    - 패딩의 두 가지 주요 목적을 요약:
        1.  피처 맵 크기의 급격한 축소 방지.
        2.  모서리 영역의 특징 정보가 충분히 반영되도록 함.

    - 새로운 섹션 "필터와 커널(Filter and Kernel)" 시작.
    - CNN에서 '필터'와 '커널'은 상호 교환적으로 사용되지만, 엄밀히 말해 필터는 여러 개의 커널(채널)로 구성됨.
    - 그림은 필터가 3D 구조이며, 내부에 여러 채널(커널)이 있음을 보여줌.

    - 필터와 커널의 3차원적 특성을 강조.
    - 그림에서 붉은색 2x2 영역을 예시로 들어, 이것이 하나의 채널 또는 커널임을 설명. 여러 개의 채널이 모여 하나의 필터를 구성.

- **04:00 - 04:30 (화면 캡처: 필터와 커널 슬라이드 - 커널 개념 재강조)**
    - 필터는 3차원(가로, 세로, 깊이)이고, 커널은 그 필터 안의 2차원적인 조각(예: 2x2 행렬)임을 재확인.

- **04:30 - 05:00 (화면 캡처: 커널 사이즈 슬라이드 - 3x3, 5x5, 7x7)**
    - "커널 사이즈(Kernel Size)" 섹션 시작.
    - 커널 크기(예: 3x3, 5x5, 7x7)는 필터의 가로x세로 크기를 의미. 일반적으로 가로와 세로가 같은 크기를 가짐.
    - 커널 크기가 클수록 한 번에 더 많은 입력 피처 정보를 가져올 수 있음.

- **05:00 - 05:30 (화면 캡처: 커널 사이즈 슬라이드 - 설명 텍스트)**
    - 커널 크기 선택의 중요성 및 장단점 설명.
    - 큰 커널은 많은 정보를 가져오지만, 연산량과 파라미터 수가 늘어나고 출력 피처 맵 크기가 크게 줄어드는 단점이 있음.
    - 학생들에게 커널 크기에 따른 차이점을 스스로 추론해 볼 것을 권유.

- **05:30 - 06:00 (화면 캡처: 커널 사이즈 슬라이드)**
    - 학생들의 상상력과 추론 능력을 독려하며, 정답보다는 사고 과정의 중요성을 강조.

- **06:00 - 07:00 (화면 캡처: 커널 사이즈 슬라이드)**
    - 학생들에게 잠시 토론 시간을 제공하며, 커널 사이즈의 영향에 대해 고민해 볼 기회를 줌.

- **07:00 - 07:30 (화면 캡처: 커널 사이즈 슬라이드)**
    - 커널 크기가 클수록 원본 이미지에서 더 많은 피처 정보를 가져올 수 있음을 확인.

- **07:30 - 08:00 (화면 캡처: 커널 사이즈 슬라이드)**
    - 큰 커널이 정보를 많이 가져오는 것은 좋지만, 이로 인해 피처 맵의 크기가 급격히 줄어드는 문제가 발생함을 언급.
    - 3x3 커널을 사용해도 사이즈 축소는 일어남.

- **08:00 - 08:30 (화면 캡처: 커널 사이즈 슬라이드)**
    - 과거 초기 CNN(예: ResNet)에서는 7x7 같은 큰 커널을 사용했으나, 이는 엄청난 연산량과 함께 출력 이미지 크기를 극심하게 줄이는 부작용이 있었음.

- **08:30 - 09:00 (화면 캡처: 커널 사이즈 슬라이드 - Modern CNNs)**
    - 현대 CNN에서는 이러한 문제 때문에 큰 커널을 사용하지 않고, 3x3과 같은 작은 커널을 선호함을 설명.

- **09:00 - 09:30 (화면 캡처: 커널 사이즈 슬라이드 - 단점 요약)**
    - 큰 커널 사용의 단점 재정리:
        1.  연산량 및 파라미터 수 증가.
        2.  최종 피처 맵 크기가 너무 작아짐.
    - 따라서 최근에는 작은 커널을 여러 층 쌓는 방식이 주류가 됨.

- **09:30 - 10:00 (화면 캡처: 커널 사이즈 슬라이드 - 예시 다이어그램)**
    - 5x5 입력 이미지에 3x3 커널을 적용하여 3x3 출력 이미지를 얻는 과정을 다이어그램으로 시각화.
    - 이 과정을 통해 이미지 크기 축소 현상을 다시 한번 설명.

- **10:00 - 10:30 (화면 캡처: 커널 사이즈 슬라이드 - 다층 컨볼루션)**
    - 다층 컨볼루션(Layer 1, Layer 2, Layer 3)의 개념 설명.
    - 한 층의 출력이 다음 층의 입력이 되며, 반복적인 컨볼루션으로 인해 피처 맵의 크기가 계속 줄어듦.

- **10:30 - 11:00 (화면 캡처: 커널 사이즈 슬라이드 - 최종 출력 1x1)**
    - 5x5 이미지가 3x3 커널 두 번의 컨볼루션을 거쳐 1x1 크기로 줄어드는 예시를 제시.
    - 이는 이미지의 특징이 최종적으로 압축되는 과정을 보여줌.

- **11:00 - 11:30 (화면 캡처: 커널 사이즈 슬라이드 - 최종 출력 확인)**
    - 강사님은 학생들이 5x5에서 3x3, 그리고 다시 1x1로 줄어드는 과정을 이해했는지 확인.

- **11:30 - 12:00 (화면 캡처: 커널 사이즈 슬라이드)**
    - 학생들에게 잠시 토론 시간을 주어 개념을 숙지할 기회를 제공.

- **12:00 - 13:00 (화면 캡처: 커널 사이즈 슬라이드)**
    - 계속해서 학생들이 서로 토론하며 이해도를 높일 수 있도록 유도.

- **13:00 - 13:30 (화면 캡처: 필터를 적용 후 이미지 특징 추출 슬라이드)**
    - "필터를 적용 후 이미지 특징 추출" 섹션 시작.
    - 필터는 마치 '렌즈'처럼 이미지의 다양한 특징(윤곽, 질감, 색상 등)을 추출하는 역할을 한다고 비유.
    - 다양한 필터를 적용하면 원본 이미지에서 여러 가지 특징을 강조한 피처 맵(Feature Map)들이 생성됨.

- **13:30 - 14:00 (화면 캡처: 필터를 적용 후 이미지 특징 추출 슬라이드)**
    - 생성된 피처 맵들은 결국 특정 객체(예: 새)를 '판별(classification)'하는 용도로 사용됨.
    - AI는 이 피처 맵들로부터 객체의 '특이점(distinctive features)'을 학습하여 새로운 이미지가 들어왔을 때 해당 객체를 식별할 수 있음.

- **14:00 - 14:30 (화면 캡처: 필터를 적용 후 이미지 특징 추출 슬라이드)**
    - 인간의 뇌가 자연스럽게 진화하여 특징을 파악하는 방식과 AI의 유사성을 언급하며, 이러한 학습 방식의 경이로움을 강조.

- **14:30 - 15:00 (화면 캡처: 필터를 적용 후 이미지 특징 추출 슬라이드)**
    - CNN이 인간의 인지 과정을 모방하여 이미지 특징을 학습하는 방식에 대한 배경 설명.

- **15:00 - 15:30 (화면 캡처: 필터를 적용 후 이미지 특징 추출 슬라이드)**
    - 다양한 렌즈(필터)를 통해 특정 특징들(특이점)만 추출한 피처 맵들이, 궁극적으로 특정 데이터(새)를 판별하는 '키(key)'가 된다는 점을 재강조.

- **15:30 - 16:00 (화면 캡처: Feature Map 슬라이드 - 전반적인 CNN 구조)**
    - CNN의 핵심적인 구성 요소(필터, 피처 맵)에 대한 설명을 마치며, CNN의 기본적인 이해가 완료되었음을 선언.

- **16:00 - 16:30 (화면 캡처: Feature Map 슬라이드)**
    - 강의 진행 방식에 대한 설명: 복잡한 내용을 처음부터 깊이 다루기보다, 핵심적인 개념들을 먼저 이해시키고자 함.

- **16:30 - 17:00 (화면 캡처: Feature Map 슬라이드)**
    - 학습의 최종 목표는 '왜' 이러한 기술을 배우는지, '무엇을 위해' 하는지에 대한 큰 그림을 이해하는 것임을 강조.

- **17:00 - 17:30 (화면 캡처: Feature Map 슬라이드)**
    - 학생들의 편안하고 효과적인 학습을 위해 오전에는 이론, 오후에는 코드를 다루는 수업 방식에 대해 언급.

- **17:30 - 18:00 (화면 캡처: Feature Map 슬라이드 - Feature Map 다이어그램)**
    - CNN은 단순히 하나의 레이어로 끝나지 않고, 여러 개의 컨볼루션 레이어(Convolutional Layer)와 피처 맵(Feature Map)으로 구성된 '여러 망(Multi-layer network)'임을 설명.

- **18:00 - 18:30 (화면 캡처: Feature Map 슬라이드 - MLP 연결)**
    - 컨볼루션 레이어를 통해 추출된 피처 맵들은 최종적으로 풀 커넥티드(Fully Connected, 선형) 레이어와 소프트맥스(Softmax) 활성화 함수를 거쳐 분류 결과(예: 몇 퍼센트의 확률로 새인지)를 출력함.
    - 이는 앞에서 배운 MLP(Multi-Layer Perceptron)가 CNN의 마지막 단에 활용됨을 의미.

- **18:30 - 19:00 (화면 캡처: 필터 값 결정 슬라이드 - Sliding Window)**
    - "필터 값 결정(Filter Value Determination)" 섹션 시작.
    - AI가 필터 값을 '스스로(스스로)' 만들고 최적화함을 강조. 인간이 직접 필터의 수치를 정하는 것이 아님.

- **19:00 - 19:30 (화면 캡처: 필터 값 결정 슬라이드 - AI의 자율 학습)**
    - AI는 학습 과정을 통해 스스로 필터의 적합성을 평가하고('이게 맞나?' 생각하고), 필요한 경우 새로운 필터를 생성하여 적용함.
    - 예를 들어, 새의 형체 필터를 만든 후, 주둥이(입 모양) 특징이 더 중요하다고 판단하면 주둥이 전용 필터를 만듦.

- **19:30 - 20:00 (화면 캡처: 필터 값 결정 슬라이드)**
    - 이러한 AI의 자율 학습은 새의 깃털 모양 등 더욱 미세한 특징까지 파악하기 위해 계속될 수 있음을 설명.

- **20:00 - 20:30 (화면 캡처: 필터 값 결정 슬라이드)**
    - AI가 '최적의 필터 값'을 학습을 통해 '스스로 최적화'하는 것이 컨볼루션의 핵심임을 재강조.

- **20:30 - 21:00 (화면 캡처: 필터 값 결정 슬라이드)**
    - 인간은 AI가 학습할 수 있는 환경(데이터, 패딩, 스트라이드, 손실 함수, 최적화 함수 등)을 설정할 뿐, 필터 값의 세부 계산은 AI가 수행함.
    - 이전 수업에서 배운 엔트로피(손실 함수), 아담(최적화 함수) 등이 여기서 활용됨을 상기시킴.

- **21:00 - 21:30 (화면 캡처: 필터 값 결정 슬라이드)**

### 3. 핵심 개념 및 코드/공식 정리

#### 1. 컨볼루션 연산 (Convolutional Operation)
- **정의**: 이미지의 특징을 추출하기 위해 작은 필터(커널)가 입력 이미지를 순회하며 각 영역의 픽셀 값과 가중치를 곱하고 더하는 연산.
- **결과**: 원본 이미지의 특징을 요약한 '피처 맵(Feature Map)'을 생성.
- **문제점**: 패딩이 없을 경우, 컨볼루션 연산 후 피처 맵의 크기가 급격히 줄어들어 공간 정보 손실 발생.

#### 2. 필터 (Filter) 및 커널 (Kernel)
- **필터**: CNN에서 이미지 특징을 추출하는 3차원적 가중치 행렬. 여러 개의 2차원 커널(채널)로 구성될 수 있음.
- **커널 (Kernel/Channel)**: 필터를 구성하는 2차원 행렬로, 특정 특징(예: 수직선, 수평선)을 감지하는 가중치 집합.
- **커널 사이즈 (Kernel Size)**: 커널의 가로x세로 크기(예: 3x3, 5x5, 7x7).
    - **큰 커널**: 한 번에 넓은 영역의 정보를 가져올 수 있지만, 연산량이 많고 출력 피처 맵 크기가 크게 줄어드는 단점. (과거 초기 CNN에서 사용).
    - **작은 커널**: 효율적인 연산과 피처 맵 크기 유지에 유리. (현대 CNN에서 선호).

#### 3. 패딩 (Padding)
- **정의**: 컨볼루션 연산 시 입력 이미지 주변에 추가적인 픽셀(주로 0)을 채워 넣어 이미지 크기를 확장하는 기술.
- **주요 목적**:
    1.  **피처 맵 크기 유지**: 컨볼루션 연산으로 인한 피처 맵 크기의 급격한 축소를 방지하여 공간 정보를 보존.
    2.  **모서리 특징 강화**: 이미지의 가장자리(모서리) 픽셀들이 필터 연산에 여러 번 참여하게 함으로써, 해당 영역의 중요한 특징 정보가 피처 맵에 더 잘 반영되도록 함. (패딩이 없으면 모서리 픽셀은 한두 번만 처리되어 정보 손실 발생).
- **패딩 수준**: `padding=1`은 한 줄의 0을, `padding=2`는 두 줄의 0을 추가.

#### 4. 스트라이드 (Stride)
- **정의**: 필터(슬라이딩 윈도우)가 입력 이미지를 가로지르며 이동하는 간격(step size).
- **종류 및 영향**:
    - **Stride 1 (기본값)**: 필터가 한 픽셀씩 이동. 피처 맵의 크기가 상대적으로 크게 유지되며, 상세한 특징 정보를 촘촘하게 추출.
    - **Stride 2**: 필터가 두 픽셀씩 이동. 피처 맵의 크기가 더 작아지고, 정보 추출은 띄엄띄엄 이루어짐.
- **효과**:
    - **연산 속도 향상**: Stride가 클수록 이동 횟수가 줄어들어 연산 속도가 빨라짐.
    - **피처 맵 크기 감소**: Stride가 클수록 출력 피처 맵의 크기가 줄어듦. (강사님께서 이전에 Stride 1이 더 줄어든다고 잘못 설명했음을 정정. Stride 2가 더 많이 줄어듦).
    - **불필요한 특징 제거**: 이미지에 노이즈나 불필요한 특징이 많을 경우, Stride를 크게 하여 이러한 특징들을 걸러내는 효과도 얻을 수 있음.

#### 5. AI의 필터 값 자율 학습
- **핵심**: 딥러닝 CNN은 사용자가 필터 값을 직접 설계할 필요 없이, 학습을 통해 스스로 최적의 필터 값을 결정하고 최적화함.
- **과정**: AI는 이미지 분류 목적에 부합하는 필터들을 스스로 만들고, 평가하며, 개선하는 과정을 반복함. (예: 새의 주둥이 특징을 더 잘 잡는 필터를 스스로 생성).
- **인간의 역할**: 인간은 AI가 학습할 수 있는 환경(입력 크기, 커널 크기, 스트라이드, 패딩 등)을 설정하며, AI는 이 설정 내에서 최적의 필터 값을 찾아냄.
